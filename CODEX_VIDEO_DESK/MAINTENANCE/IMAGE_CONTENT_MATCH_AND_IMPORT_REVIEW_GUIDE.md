# 그림 내용(CLIP) 매칭 + 패널 import 검수 가이드 (A 설계)

목표: 일러스트 재사용/배정을 **파일명·태그가 아니라 "그림이 실제로 무엇을 그렸는가(픽셀
내용)"** 로 판단한다. 그래서 (1) 라이브러리를 모아 재활용할 때 이름이 틀려도 오염되지
않고, (2) GPT 그림을 가져올 때 사람이 일일이 파일명을 맞추지 않아도 된다.

이 문서는 종민의 질문에서 출발했다:
> "1번(자동배정) 하면 옮길 땐 편한데, 나중에 일러스트 폴더 모아놓고 재활용할 때 문제 있지 않아?"

답: 맞다. 이름 기반 재사용 + 부정확한 자동배정은 **라이브러리를 영구 오염**시킨다.
그래서 매칭 기준 자체를 "그림 내용"으로 바꾸고, 이름을 정하는 단계(import)는 사람이
패널 안에서 썸네일·신뢰도를 보고 확정하게 했다.

---

## 1. 무엇이 바뀌나 (한눈에)

기존: 버튼 2 = 다운로드 그림을 **시간순(mtime)으로 추측** 매핑 → 워커가 자동 "Y" →
       이름이 틀려도 그대로 라이브러리에 들어감(조용한 오염 위험).

지금: 버튼 2 = **그림 내용으로 자동 배정 "제안"** → 패널에 썸네일+신뢰도+중복경고 표시
       → 사람이 확인/수정 후 **확정** → 그 확정 매핑대로만 들어감 → 그림 지문 등록 →
       렌더가 **그림 내용 기준**으로 재사용.

핵심 모델: `jinaai/jina-clip-v1` — 이미지와 텍스트(한국어 포함)를 **같은 768차원 공간**에
임베딩한다. 그래서 한국어 청크 텍스트 ↔ 그림 픽셀의 교차매칭이 된다.

---

## 2. 새 파일 / 바뀐 파일

신규
- `shorts/scripts/codex_image_embed.py` — 그림·텍스트 임베딩 엔진. 라이브러리 그림 지문을
  `shorts/codex/image_embed_cache.json` 에 파일 mtime/size 기준으로 캐시. 모델 없으면
  `available()=False` 로 폴백(태그 DB 는 절대 건드리지 않음).
- `shorts/scripts/codex_import_propose.py` — 다운로드 후보를 그림 내용으로 요청에 배정
  "제안"만 한다(파일 미이동). 결과 = `CODEX_VIDEO_DESK/IMPORT_PROPOSAL.json`.

바뀜
- `shorts/scripts/codex_import_downloads.py` — `IMPORT_CONFIRMED.json`(패널 확정 매핑)이
  있으면 **그대로** 복사하고 그림 지문을 캐시에 채운다. 없으면 기존 동작(폴백).
- `shorts/scripts/codex_semantic_visual_match.py` — 확신 있는 텍스트 매칭이 없을 때,
  **중립 필러 대신** 그림 내용(CLIP) 매칭으로 채운다. (잘 맞던 매칭을 덮어쓰지 않음 →
  기존 품질 회귀 위험 없음.)
- `CODEX_VIDEO_DESK/dashboard/server.py` — GET `/api/illust-thumb`(썸네일),
  POST `video_import_propose`(제안), POST `video_import_confirm`(확정→대기열). 버튼 2 가
  검수 패널을 연다. **패널을 껐다 켜야 적용됨**(서버 코드 변경).
- `shorts/SETUP_EMBED.bat` — 텍스트 모델 + jina-clip 이미지/텍스트 모델 + pillow 까지 설치·검증.

## 3. 작업 흐름 (사람이 하는 것)

1. 패널 버튼 **1. 영상용 프롬프트 준비** → 일러스트 요청서/프롬프트 생성.
2. GPT Plus 로 그림을 그려 **다운로드**(파일명 아무거나 OK. ILLUSTRATION_DROP 또는 Downloads).
3. 패널 버튼 **2. 이미지 가져오기 + 렌더** → "이미지 가져오기 검수" 패널이 열림.
   - 각 그림의 썸네일 + 자동 배정된 파일명(드롭다운) + 신뢰도 + 중복경고가 보인다.
   - 틀린 게 있으면 드롭다운으로 바꾸고, 안 쓸 그림은 "사용 안 함"으로.
4. **확정 → 가져오고 렌더** 클릭 → 확정 매핑이 기록되고 렌더 대기열에 등록된다.
   별도 실행파일/콘솔 없음. 전부 패널 안에서 끝난다.

확정 단계에서 사람이 파일명을 검수하므로, **틀린 이름이 라이브러리에 들어가는 일이 차단**된다.
그리고 렌더 재사용은 그림 "내용"으로 하므로, 혹시 이름이 살짝 어긋나도 내용으로 맞춘다(이중 안전).

---

## 4. 설치 / 켜기

각 PC에서 1회(온라인 필요):
```
shorts\SETUP_EMBED.bat
```
- `[RESULT] Image-content matching ENABLED (TRUE)` 가 나오면 켜진 것.
- 모델은 첫 실행에 한 번만 받고 이후 오프라인 동작. 총 ~1GB(text 0.55GB + image 0.34GB).
- 패널 서버 코드가 바뀌었으니 **패널을 껐다 켠다**.

모델이 없거나 설치 실패해도 파이프라인은 안 깨진다:
- 제안: 시간순 추정 매핑으로 폴백(패널에서 고치면 됨).
- 렌더: 기존 텍스트/lexical 매칭으로 폴백.

---

## 5. 임계값 튜닝 (환경변수)

- `PHONESPOT_IMG_MATCH_MIN` (기본 `0.28`) — 렌더에서 그림 내용 매칭을 채택하는 최소 코사인.
  교차모달 코사인은 텍스트끼리(MiniLM)보다 절대값이 낮다. PC에서 렌더 결과를 보고
  올리면(엄격, 무관 매칭↓) / 내리면(관대, 빈 슬롯↓).
- `PHONESPOT_CLIP_MODEL` (기본 `jinaai/jina-clip-v1`) — 이미지/텍스트 공용 CLIP 모델.
- 제안 단계 중복 경고: `codex_import_propose.py` 의 `DEDUP_WARN`(기본 0.93).

---

## 6. PC 검증 체크리스트

1. `SETUP_EMBED.bat` → 이미지 매칭 ENABLED 확인.
2. 패널 재시작 후 버튼 2 → 검수 패널에 썸네일과 신뢰도가 보이는지.
   - 엔진 표시가 "그림 내용 매칭" 이어야 함("시간순 추정"이면 모델 미설치).
3. 일부러 파일명을 안 맞춘 그림을 넣어 보고, 자동 배정이 **내용대로** 맞는지 눈으로 확인.
4. 확정 → 렌더 → 결과 영상에서 그림이 청크 내용과 맞는지.
5. 매칭이 과하게 엄격/관대하면 `PHONESPOT_IMG_MATCH_MIN` 조절.

---

## 7. 정직한 한계 / 주의

- **교차모달 품질은 PC에서만 검증 가능**: 개발 샌드박스는 모델 다운로드가 막혀 있어
  로직(폴백/캐시/배정/보안)만 mock 으로 검증했다. 한국어↔그림 실제 매칭 정확도는
  종민의 PC에서 위 체크리스트로 확인해야 한다.
- 그림 내용 매칭은 **확신 텍스트 매칭이 없을 때만** 보조로 쓴다 → 기존 매칭을 덮어쓰지
  않으므로 회귀 위험은 없다. 다만 임계값이 너무 낮으면 약한 매칭이 늘 수 있다.
- 확정 매핑(`IMPORT_CONFIRMED.json`)은 24시간·동일 slug 일 때만 사용하고, 쓰고 나면
  `.used.json` 으로 치워 재사용을 막는다(다른 영상에 잘못 적용 방지).
- 썸네일 GET 은 DROP/Downloads/illustrations 안의 이미지 파일만 제공한다(경로 탈출 차단).

## 8. 패널 코드를 바꾼 뒤 재시작(껐다 켜기) 없이 반영하기

무엇을 바꿨느냐에 따라 다르다.

- **스크립트(`shorts/scripts/codex_*.py`, 프롬프트 문구 등)**: 패널이 버튼을 누를 때마다
  새 프로세스로 실행하므로 **재시작 불필요**. 파일만 바뀌면 다음 클릭에 바로 반영된다.
  (프롬프트 문구는 버튼 1 을 다시 눌러 LATEST_PROMPT.md 를 새로 만들면 반영.)
- **패널 본체(`dashboard/server.py` = 웹 UI/JS/API)**: 계속 떠 있는 파이썬 프로세스라
  메모리에 올라간 코드를 핫리로드하지 못한다 → 프로세스를 다시 띄워야 한다.

수동으로 끌 필요는 없다. 런처 `dashboard/start_hidden.ps1` 가 **버전 게이트**로 자동
재시작한다:
- `/api/health` 의 버전이 런처가 기대하는 `$expectedVersion` 과 다르면, 옛 패널을 죽이고
  새로 띄운다.

그래서 **server.py 를 바꿀 때 규칙**:
1. `server.py` 의 `PANEL_VERSION` 을 한 칸 올린다. 예: `phonespot-web-v6` → `phonespot-web-v7`.
2. `start_hidden.ps1` 의 `$expectedVersion` 을 **똑같이** 올린다.
3. `00_PHONE_SPOT_PANEL.bat` 을 다시 더블클릭 → 버전 불일치 감지 → 옛 패널 자동 종료 →
   새 코드로 자동 재시작.
4. 브라우저 탭 새로고침(캐시까지: Ctrl+Shift+R).

두 값을 같이 올리지 않으면:
- 둘 다 그대로면 런처가 "정상"으로 보고 **재시작을 건너뛰어 새 코드가 안 올라온다**(가장 흔한 실수).
- 한쪽만 올리면 매번 불필요하게 재시작한다.

현재 버전: `phonespot-web-v6` (이미지 검수 기능 반영본).
