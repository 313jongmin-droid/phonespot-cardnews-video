# 멀티 PC: 완전 독립 로컬 생산기 + 일러스트 라이브러리 공유

## 결정 (2026-06)

- 멀티 PC = **각 PC가 완전 독립 로컬 생산기**. 카드 수집 → 영상 준비 → 렌더 → 출력을
  전부 자기 PC 로컬에서 한다. **중앙 서버·원격 렌더 큐에 의존하지 않는다.**
- **원격 렌더 큐/워커 방식은 접는다**(반쪽짜리였고, 방화벽/인터넷/edge-tts 의존으로 깨졌음 —
  실패 로그: 원격 PC 에서 pip edge-tts 설치가 WinError 10013 으로 차단됨).
- 공유는 **옵션**이며, 만드는 건 **일러스트 라이브러리 공유 하나**다(수렴 목표에 필수).
  결과물/작업분배 공유는 지금 안 만듦(필요하면 나중에).

근거: 메인 PC 는 이미 로컬에서 전 과정을 다 한다. 다른 PC도 풀 설치 + 패널이면 똑같이 독립
동작한다. 그러면 큐·워커·패키지 전송·패널 노출·방화벽 의존이 전부 사라진다(가장 단순·안정).

## A. 새 PC 를 독립 생산기로 만들기 (1회)

### 권장: 원샷 설치
1. 코드 받기(저장소가 없으면): `00_SETUP_RENDER_PC_FROM_GITHUB.bat` (git clone).
   이미 코드가 있으면 건너뜀.
2. **`SETUP_FULL_PRODUCER.bat` 더블클릭** — 한 번에 다 깔고 검증합니다:
   - npm install(Remotion) + python deps(edge-tts/pillow/mutagen/requests/playwright/**flask**/fastembed/numpy)
   - playwright chromium + 임베딩 모델(text + CLIP image) 1회 다운로드
   - 마지막에 `codex_producer_check.py` 로 **PASS/FAIL 점검표** 출력.
3. `00_PHONE_SPOT_PANEL.bat` 실행. 카드 생성은 `cardnews\webui\start.bat`.

점검만 따로: `shorts\scripts\codex_producer_check.py` (치명 항목 0이면 준비 완료).

### 수동(단계별)도 가능
`00_SETUP_RENDER_PC_FROM_GITHUB.bat` → `shorts\SETUP_EMBED.bat` → (Flask 설치) → 패널 실행.

이러면 그 PC 는 인터넷이 막혀도(모델·deps 설치만 끝났으면) 혼자 생산·렌더가 됩니다.

> 중앙서버 동기화(`01_SYNC_CARDNEWS_*`, `\\192.168.0.7`)는 **옵션**입니다. 메인 PC 의
> 카드뉴스를 당겨오고 싶을 때만 쓰고, 완전 독립이면 안 써도 됩니다. 패널의 "카드뉴스 동기화"
> 버튼은 그 공유가 있을 때만 의미가 있습니다(없으면 그 버튼만 실패할 뿐, 로컬 생산엔 무관).

체크: 각 PC 는 렌더를 자기 로컬 워커로 처리하므로 다른 PC 를 켜둘 필요가 없습니다.

## B. 일러스트 라이브러리 공유 (옵션, 수렴에 핵심)

왜: 안 하면 PC 마다 같은 그림을 또 그려서 수렴(생성→0)이 PC별로 따로 일어나고 중복만 는다.
라이브러리를 함께 쌓아야 "전체적으로 생성이 0 으로 수렴"이 성립한다.

### 공유 허브 정하기
공유폴더/구글드라이브/USB/네트워크 어디든 한 곳을 허브로 정하고, 각 PC 에 알려준다(둘 중 하나):
- 환경변수 `PHONESPOT_LIBRARY_SHARE=<허브경로>` (예: `setx PHONESPOT_LIBRARY_SHARE "G:\공유\phonespot_library"`)
- 또는 파일 `shorts\config\library_share_path.txt` 에 경로 한 줄.

### 실행 (둘 중 편한 방법)
- **패널 버튼(권장)**: 영상 모드 작업 목록의 **"일러스트 라이브러리 공유"** 버튼 클릭 →
  결과(가져옴/올림 개수)가 패널 하단 **실행 로그**에 표시됩니다.
- **배치 파일**: `CODEX_VIDEO_DESK\라이브러리_공유_동기화.bat` 더블클릭. (미리보기: 인자 `--dry-run`)

> 둘 다 같은 `codex_library_sync.py` 를 돌립니다. 허브 경로 설정이 먼저입니다(아래).

동작(`codex_library_sync.py`):
- 그림 `shorts/public/assets/illustrations/*.png` 을 **양방향 추가병합**.
  - 한쪽에만 있는 새 파일명만 서로 복사. **같은 파일명은 안 덮음(비파괴).**
- 태그DB `shorts/config/illustration_tag_db.json` 을 **항목 합집합**으로 병합(키워드/태그 union),
  로컬·허브 양쪽에 동일하게 기록.
- 허브 파일이 드라이브 부분동기화로 NUL 깨져도 **관대 로더**로 복원.
- 임베딩 지문은 동기화 안 함 — 새로 받은 그림은 **다음 렌더/버튼1에서 자동 재생성**.

권장 운용: 그림을 새로 확정한 뒤, 또는 하루 한 번 각 PC 에서 동기화 한 번씩. 순서 무관(양방향).

### 정직한 한계
- "비파괴 추가병합"이라, **같은 개념 파일명에 서로 다른 그림**을 그리면 각자 자기 버전을 유지합니다
  (덮어쓰기 안 함). 진짜로 한 개념=한 그림으로 수렴시키려면 파일명 규칙(CLIP 검수에서 같은 개념은
  같은 이름) 을 지키는 게 좋습니다. 충돌 파일을 강제로 한쪽으로 맞추는 기능은 일부러 안 넣었습니다(안전).
- 공유 허브가 구글드라이브면, 드라이브가 다 내려받기 전(placeholder) 상태에서 돌리면 일부 그림이
  안 보일 수 있습니다. 드라이브 동기화가 끝난 뒤 실행하세요.

## C. 라이브러리 중복 정리 (수렴 유지)

공유는 비파괴라 시간이 지나면 "같은 개념, 다른 이름" 그림이 허브에 쌓입니다. 이걸 주기적으로
정리해야 라이브러리가 비대해지지 않고 "한 개념 = 한 그림"으로 수렴합니다.

도구: `codex_library_dedup.py` — CLIP 이미지-이미지 유사도로 근접중복을 군집화.

- **점검(읽기전용, 안전)**: 패널 **"라이브러리 중복 점검"** 버튼 → 실행 로그 + 리포트
  `shorts/codex/library_dedup_report.md`(군집별 대표/중복 후보·유사도).
- **정리(삭제+병합)**: `CODEX_VIDEO_DESK\라이브러리_중복정리.bat --apply`
  - 각 군집에서 **대표 1개만 남기고** 나머지 png 삭제 + 태그DB를 대표로 병합.
  - 대표 선택: 사람이 붙인 이름(비-cpt_) 우선 → 짧은 이름 → 알파벳.
  - 임계값 `PHONESPOT_DEDUP_SIM`(기본 0.92). 낮출수록 더 많이 묶음(과합치기 주의).
- 권장 순서: ① 라이브러리 공유 동기화(백업 겸) → ② 중복 점검(리포트 확인) → ③ 필요시 `--apply`
  → ④ 다시 공유 동기화로 정리 결과 전파.
- 주의: `--apply`는 파일 삭제. 리포트로 먼저 확인하고, 허브/깃 백업이 있는 상태에서 실행.

## D. 라이브러리 백업/스냅샷 (안전망)

라이브러리는 가장 중요한 자산입니다. 실수 삭제·중복정리(--apply) 사고·드라이브 NUL 손상에 대비해
타임스탬프 스냅샷을 떠 둡니다.

도구: `codex_library_backup.py`
- 패널 **"라이브러리 백업"** 버튼 또는 `CODEX_VIDEO_DESK\라이브러리_백업.bat`.
- 대상: 로컬 라이브러리(일러스트+태그DB+사용이력). 허브(`PHONESPOT_LIBRARY_SHARE`) 설정 시 허브도 함께.
- 보관 위치: `PHONESPOT_LIBRARY_BACKUP`(없으면 저장소 옆 `phonespot_library_backups`). 깃/동기화 밖.
- 회전 보관: 최근 `PHONESPOT_LIBRARY_BACKUP_KEEP`(기본 10)개만 유지.
- 무결성: 0바이트/NUL 손상 의심 파일을 경고로 표시(스냅샷은 그대로 떠서 타임라인 보존).
- 복원: 스냅샷 폴더의 `illustrations/*.png` 와 json 을 원위치로 복사(수동).
- 권장: 중복정리 `--apply` 전, 그리고 주기적으로(자동화하려면 스케줄 작업으로 매일 1회 가능).

## E. 코드 업데이트 전파 (메인 개발 → 다른 PC)

코드는 **GitHub로** 도는데, 기본은 수동 2단계입니다:
- 메인(개발) PC: 패널 **"시스템 업로드"**(commit/push) → GitHub.
- 다른 PC: 패널 **"시스템 업데이트"**(pull) → 받음.

받기 전용 PC를 **시작 시 자동 업데이트**로 만들 수 있습니다:
- 그 PC에서 `2번째PC_자동업데이트_켜기.bat` 1회 실행(`PHONESPOT_AUTO_UPDATE=1` 설정).
- 이후 `00_PHONE_SPOT_PANEL.bat` 실행 때마다 `start_hidden.ps1`이 `git pull --ff-only` →
  새 코드를 받으면 패널을 자동 재시작합니다.
- **메인 PC에서는 켜지 마세요** — 작업 중 변경이 pull 로 덮일 수 있습니다(그래서 환경변수로
  받기 전용 PC만 켭니다).

주의(자동 pull 이 못 하는 것):
- **의존성 변경**(새 pip/npm 패키지)은 pull 로 설치 안 됨 → 그 PC에서 `SETUP_FULL_PRODUCER` 재실행.
- **이미 실행 중**일 때 메인이 push 하면, 그 PC는 다음 시작 때 받습니다(즉시 받으려면 "시스템 업데이트" 클릭).
- `--ff-only` 라 그 PC에 로컬 수정이 있어 분기되면 pull 이 거부됩니다(받기 전용으로만 쓰면 안전).
- 부트스트랩: 이 자동기능 자체도 코드라, 다른 PC가 **한 번은 수동 "시스템 업데이트"+재시작**으로 받아야 그다음부터 자동이 됩니다.

## 공유 안 하는 것 (의도적)
- 임베딩 캐시(image_embed_cache 등): 각 PC 자동 재생성.
- 슬러그별 산출물(shorts_script.json, audio, 카드 이미지): 각 PC 로컬.
- 최종 mp4: 지금은 공유 안 함. 원하면 단순 공유폴더로 따로 모으면 됨(별도 옵션).
- 작업 중복방지(누가 어느 슬러그): 지금 안 만듦(구글시트 큐마스터 미적용분 있음, 필요시).

## 안 쓰게 된 것
- 원격 렌더 큐/워커(`RENDER_WORKER`, `video_import_render` 원격 경로). 코드는 남아있지만
  독립 운영에선 쓰지 않는다. (패널 로컬 워커는 자기 PC 렌더용으로 계속 사용.)
