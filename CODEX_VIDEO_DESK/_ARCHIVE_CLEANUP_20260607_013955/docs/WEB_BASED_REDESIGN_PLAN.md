# PhoneSpot 00 Panel Web-Based Redesign

## 원칙

- 새 패널을 만들지 않는다.
- `00_PHONE_SPOT_PANEL.bat`로 여는 기존 `dashboard/server.py` 패널을 그대로 쓴다.
- UI, 버튼, 작업 흐름 이름은 유지한다.
- 바꾸는 것은 실행 구조다.

## 현재 구조

```text
각 PC
  00 패널 실행
  로컬/공유 폴더 읽기
  로컬에서 GitHub 다운로드/동기화
  로컬에서 Remotion 렌더
```

문제:

- 부사수 PC가 공유폴더에서 실행하면 권한 문제가 난다.
- PC마다 Git 상태와 런타임 파일이 섞인다.
- 새 카드뉴스를 보려면 동기화/새로고침 절차가 필요하다.
- 다른 PC 브라우저에서 패널을 열어도 렌더는 어느 PC에서 도는지 애매하다.

## 목표 구조

```text
메인 PC
  00 패널 서버 실행
  카드뉴스 원본 / 작업 큐 / 결과 목록 관리

부사수 PC 또는 렌더 PC
  브라우저로 메인 PC의 00 패널 접속
  렌더 워커만 로컬 실행
  작업을 가져와 로컬에서 렌더
  결과와 로그를 메인 PC 패널로 보고
```

## 1단계: 기존 00 패널을 LAN 웹 패널로 전환

`dashboard/server.py`는 기존 UI 그대로 두고 서버 바인딩만 바꾼다.

```text
기존: 127.0.0.1:4901
변경: 0.0.0.0:4901
```

내 PC:

```text
http://localhost:4901
```

다른 PC:

```text
http://메인PC_IP:4901
```

이 단계에서는 다른 PC도 같은 00 패널 화면을 볼 수 있다.

## 2단계: 버튼 동작을 작업 큐 기반으로 분리

기존 버튼은 유지한다.

```text
1. 영상용 프롬프트 준비
2. 이미지 가져오기 + 렌더
3. 선택 영상만 렌더
```

다만 내부 동작만 바꾼다.

```text
현재:
  버튼 클릭 -> 패널 서버 PC에서 바로 실행

목표:
  버튼 클릭 -> 작업 큐에 등록
  렌더 워커 PC가 작업 가져감
  워커 PC 로컬에서 실행
  로그/결과를 패널 서버로 전송
```

## 3단계: 렌더 워커

렌더 PC에는 UI가 필요 없다.

```text
렌더 워커 역할:
  서버에 등록
  대기 작업 확인
  필요한 카드뉴스 파일 다운로드
  Remotion 렌더 실행
  로그 전송
  MP4 결과 업로드
```

부사수 PC에서 필요한 것은 이것뿐이다.

```text
렌더 워커 시작
브라우저에서 메인 PC 00 패널 접속
```

## 4단계: GitHub 역할 축소

GitHub는 코드 업데이트용으로만 쓴다.

운영 데이터는 GitHub로 주고받지 않는다.

- 카드뉴스 결과물
- 이미지
- shorts_script.json
- 렌더 로그
- MP4 결과물

이 항목들은 패널 서버와 렌더 워커가 주고받는다.

## 최종 사용 흐름

```text
1. 메인 PC에서 00 패널 실행
2. 부사수 PC는 브라우저로 http://메인PC_IP:4901 접속
3. 부사수 PC에서 렌더 워커 실행
4. 패널에서 기존 버튼 그대로 클릭
5. 렌더 워커가 작업 수행
6. 기존 패널의 실행 상태/로그/결과 영역에 반영
```

## 개발 순서

1. 기존 00 패널 LAN 접속 가능하게 만들기
2. 작업 큐 저장소를 `CODEX_VIDEO_DESK/WORK_QUEUE`에 추가
3. 렌더 워커 API 추가
4. 기존 버튼의 실행 대상을 `직접 실행` 또는 `워커 큐 등록`으로 선택 가능하게 변경
5. 부사수 PC 초기 세팅 bat 추가
6. GitHub/동기화 버튼은 유지하되 운영 필수 흐름에서 제외
