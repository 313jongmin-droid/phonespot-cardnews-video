# 인사이트 누적 학습 루프 — 시트 → 클로드 (★ 2026-06-10 단순화)

> 관리대장 시트의 **유튜브·메타 성과 데이터**를 매 수집·캡션 사이클마다 자동 반영해서
> 시간이 갈수록 후보 선정·후킹 작성이 정교해지게 만드는 루프.
> 코덱스 / GitHub / Drive MD / mklink / 노트북 .bat = **전부 폐기**. 시트 직접 Read 단일 모델.

---

## 1. 전체 흐름

```
[관리대장 시트]                                  [phonespot_cardnews 폴더]
  ├─ 유튜브_인사이트 시트                          ├─ cardnews/articles/*.json
  │   (Apps Script 매일 03:40 자동)               │   (발행 완료 슬러그 = 중복방지 DB)
  ├─ 메타_인사이트 시트 ★ 추가 필요               └─ cardnews/_state/content_guide.md
  │   (현재는 Drive MD에 저장, 시트로 옮길 것)
  ├─ 유튜브 / 메타_소재 / 메타_통합 / 추세 등 원본
  └─ 동기화_로그 (Apps Script 실행 이력)
        ↓ sync_sources 매 세션 자동
[클로드 (이 세션, Cowork)]
  - 신규 수집 시: 시트 Read → 후보 가중치 계산 → 발행 슬러그 회피 → 표 생성
  - 캡션 작성 시: 시트 Read → Top 후킹·헤드라인 패턴 5채널 첫 줄에 적용
```

매 사이클마다 새로 Read = **누적 학습 자동**. 시트가 업데이트되면 다음 사이클부터 새 데이터 반영.

---

## 2. 시트 데이터 표준 (사장님이 시트 측에서 보장)

### 2-1. `유튜브_인사이트` 시트 (★ 이미 존재)

Apps Script `generateYouTubeInsights()` 가 매일 03:40 자동 갱신. 현재 포함:

- 갱신 시각 + 분석 영상 수 + 평균 조회수 + 평균 retention
- **Top 키워드** (빈도 + 평균 조회수) — 후보 가중치용
- **후킹 패턴** (숫자 포함 / 의문문 등) — 캡션 첫 줄용
- **우수 영상 공통점**
- **Top 10 조회수 영상**
- **Top 5 시청 지속률**

→ 그대로 활용. 추가 손볼 것 없음.

### 2-2. `메타_인사이트` 시트 (★ 사장님이 시트 추가해야)

현재 `generateMetaInsightsMarkdown()` 함수는 Drive MD 파일에 저장 중 (동기화_로그 `OK (11 ads / 0 campaigns)` 확인됨). → **시트로 옮겨야** 클로드가 sync_sources로 자동 Read.

필요 포맷 (유튜브_인사이트 시트 패턴과 일치):

```
🎯 메타 인사이트 — 자동 학습 결과
갱신: YYYY-MM-DD HH:MM | 분석 광고: N개 | 평균 CTR: X% | 평균 CPC: ₩XXX

★ Top 키워드 (광고 카피)
키워드,빈도,평균 CTR,평균 CPC

★ Top 5 헤드라인 패턴
패턴,예시,평균 CTR

★ Top 5 우수 광고 (CTR 기준)
광고명,헤드라인,CTR,CPC,컨셉_태그

★ 회피 키워드 (CTR 1% 미만)
키워드,평균 CTR
```

→ Apps Script 수정 필요 (시트 출력으로 변경). 사장님 직접 또는 별도 진행.

### 2-3. `cardnews/articles/*.json` (★ 이미 존재)

발행 완료된 카드뉴스 슬러그 모음. 클로드가 Glob 으로 폴더 스캔 → 토픽 중복 즉시 감지.

## 3. 가중치 룰 (★ 수집 단계)

후보 표 각 항목에 다음 라벨 부착:

| 라벨 | 조건 | 가중치 |
|---|---|---|
| `yt+30%` | 후보 토픽 키워드가 유튜브_인사이트 Top 10 키워드 매칭 | +30% |
| `yt-hook+20%` | 토픽이 유튜브 Top 5 후킹 패턴 (의문문·숫자포함 등) 적용 가능 | +20% |
| `meta+20%` | 메타_인사이트 Top 헤드라인 패턴 매칭 | +20% |
| `meta+30%` | 메타 우수 광고 (상위 5) 컨셉 직접 매칭 | +30% |
| `season+30%` | 매뉴얼 시즌 핫토픽 (caption_template.md §6 시즌) 매칭 | +30% |
| `store-active+30%` ★ | **매장이 현재 광고 운영 중인 캠페인** = 시트 (메타·당근·카카오·네이버·스레드·인스타) 최근 7일 운영 메모에서 키워드 추출. 매장 자체가 직접 푸시 = 정합 최강 | +30% |
| `dup-100%` | `cardnews/articles/` 에 동일 토픽 이미 발행 → **즉시 제외** | -100% (강제) |
| `회피-100%` | 매뉴얼 회피 키워드 (통합요금제·요금 인하 등) | 표기 후 사장님 판단 |
| `freshness+10%` | 발행일 D-7 이내 검증된 신선 | +10% |

후보 종합 점수 = 기본 100% + 가중치 합산. 표에 점수 또는 라벨 모두 표기.

## 4. 가중치 룰 (★ 캡션 작성 단계)

매 카드뉴스 JSON 작성 시 시트 다시 Read → 5채널 첫 줄 후킹 작성에 적용:

### 4-1. 유튜브 Top 후킹 패턴 우선 적용

예시 (유튜브_인사이트 시트 현재 데이터):
- 숫자 포함: "가입자 1,100만 명 유심 전체 교체하는 LG유플러스"
- 의문문: "언제까지 기다려? 애플페이 근황"

→ 카드뉴스 유튜브 채널 (channel 4) 첫 줄 = 이 패턴 우선

### 4-2. 메타 Top 헤드라인 패턴

메타 광고가 잘 먹은 카피 톤 → 인스타·틱톡 등 짧은 후킹 채널 적용

### 4-3. 회피 자가 검증

- 매뉴얼 회피 키워드 0건
- 시트 회피 키워드 (CTR 1% 미만) 0건
- "안양" 0건

## 5. 클로드 매 사이클 자기 검증 체크리스트

### 수집 시작 시
- [ ] 시트 (유튜브_인사이트 + 메타_인사이트) Read 완료?
  - 못 읽으면 1줄 명시: "시트 sync 안 됨 → 가중치 미적용"
- [ ] `cardnews/articles/*.json` Glob 으로 기존 슬러그 추출?
- [ ] 인사이트 갱신 시각 vs 오늘 = 신선도 판단?
- [ ] ★ **시트 채널 운영 시트 (메타·당근·카카오·네이버·스레드·인스타) 최근 7일 "진행사항/수정" 메모 스캔**? → 매장 현재 진행 캠페인 키워드 추출 (예: "삼성 페스티벌 광고 등록", "S26 사전예약 소재 등록" 등) → `store-active+30%` 후보 강제 추가
- [ ] WebSearch 4 라인 키워드에 매장 진행 캠페인 키워드 반영?

### 후보 표 출력 시
- [ ] 각 후보에 가중치 라벨 부착?
- [ ] 중복 토픽 자동 제외?
- [ ] 회피 키워드 표기 (사장님 판단용)?

### 캡션 작성 시
- [ ] 유튜브 Top 후킹 패턴 = 유튜브 채널 첫 줄에 적용?
- [ ] 메타 Top 카피 패턴 = 짧은 채널 (인스타·틱톡) 적용?
- [ ] 회피 키워드 0건?
