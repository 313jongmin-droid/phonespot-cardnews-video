# .bat 파일 작성·수정 룰 (★ 2026-05-31 사고 후 신설)
- **금지 1 — 한글 주석·메시지**: `.bat` 안 한글 들어가면 Windows CP949 환경에서 mojibake (`??뚯씪` 식). `rem`, `echo`, 로그 메시지 **모두 영문 only**
- **금지 2 — LF line ending**: `Write` 도구는 `.bat`을 LF로 저장 → cmd가 `goto` 라벨 인식 못 함 (`'select_mode'은(는) 내부 또는 외부 명령...` 에러)
- **필수 절차** — `.bat` Write/Edit 후 **즉시 bash로 CRLF 변환**:
  ```python
  data = open(p, 'rb').read()
  data = data.replace(b'\r\n', b'\n').replace(b'\n', b'\r\n')
  open(p, 'wb').write(data)
  ```
- **즉시 검증**: `file *.bat` 출력에 `CRLF line terminators` 포함 확인
- **재발 차단 사례**: run_pngs.bat 셀렉트 메뉴 추가 시 한글 주석 + LF 저장 → 사장님 PC에서 라벨 인식 실패. 위 룰로 차단

# JSON 작성/수정 룰 (★ 2026-05-29 사고 후 신설)
- **새 JSON 파일 작성**: `Write` 한 번에 통째로. f-string · 수동 string concat 금지
- **기존 JSON에 필드 추가 / large multi-line value 변경**: `Edit` 도구 사용 금지. 대신 bash + Python:
  ```python
  import json
  data = json.loads(open(fp, encoding='utf-8').read())
  data['new_field'] = '...'
  with open(fp, 'w', encoding='utf-8') as f:
      json.dump(data, f, ensure_ascii=False, indent=2)
  json.load(open(fp, encoding='utf-8'))  # 즉시 re-read 검증
  ```
- **작성/변경 직후 항상 `json.load()` 재검증** — JSONDecodeError 즉시 잡힘
- **사고 사례**: Edit으로 `narration_md` 추가 시 new_string 마지막 `}` 미반영 → JSON 파손. 위 룰로 재발 차단

# 자동화 흐름

> ⚠ **「수집」 단계는 AI 대화 단계입니다.** 실행 가능 `.bat`/`.py` 스크립트 없음. 사장님이 클로드 대화창에서 `신규 수집` 트리거 → AI가 WebSearch + WebFetch 도구로 자동 수집 → 후보 표 제출 → 사장님 셀렉 → AI가 JSON·prompt.md·폴더 동시 작성. **dashboard·외부 도구에서 "수집" 직접 트리거 불가**. 진짜 자동 수집이 필요하면 Anthropic API 통합 별도 Phase. 따라서 dashboard의 "수집" 버튼은 사실상 **"후보 새로고침"** (이미 작성된 articles/images/output 스캔) 역할만.

## 트리거 방법 (★ 2026-06-03 갱신 — 두 경로 모두 OK)

### 방법 1: 클로드 앱 직접 (현재 주력)

### 방법 2: 텔레그램 트리거 (★ 2026-06-03 구현 완료)
- **위치**: 사장님 폰 텔레그램 → 폰스팟 봇
- **명령**: `신규 수집` / `사기 토픽` / `꿀팁 토픽` / `Q&A 토픽` / 또는 영문 `news`·`scam`·`tip`·`qa`
- **흐름**:
  1. 사장님 폰에서 봇에 명령 메시지
  2. `automation/scripts/telegram_listener.py` (PC daemon) 가 long polling으로 수신
  3. 명령 검증 (trusted chat_id) + 회신 ("명령 수신, PC 진행 시작")
  4. 명령어를 **PC 클립보드에 복사**
  5. **Claude 데스크탑 앱 자동 실행** (foreground)
  6. **모드 선택**:
     - `manual`: 사장님 PC에서 Ctrl+V + Enter (기본)
     - `auto`: pyautogui로 자동 입력 (`pip install pyautogui` 필요)
  7. 결과 회신 (클립보드/실행/입력 OK/FAIL)
- **실행**: `automation\start_telegram_listener.bat` (Windows Task Scheduler 등록 권장 — 부팅 시 자동)
- **설정 파일** (`_secrets/listener_config.json`, 선택):
  ```json
  {
    "mode": "manual",
    "claude_path": "C:/Users/.../Programs/Claude/Claude.exe",
    "trusted_chat_ids": ["사장님_chat_id"]
  }
  ```
  파일 없으면 기본값 (manual 모드 + `_secrets/telegram_chat_id.txt` 단일 chat_id)
- **완전 자동 (텔레그램 한 번으로 끝)** = `mode=auto` + Claude 앱이 항상 떠 있으면 사실상 완결. 단 첫 셋업 + pyautogui 설치 필요

### 방법 3: outbox 자동 푸시 (★ 2026-06-03 신설, PC→폰 송신)
- **목적**: 클로드가 신규 수집 결과·검수 보고·완료 통지를 텔레그램으로 자동 전송
- **원리**: 클로드는 샌드박스 → 텔레그램 API 직접 호출 차단. 따라서 PC측 listener 가 outbox 폴더 watch + 푸시
- **사용 규약**:
  1. 클로드는 결과 텍스트를 `_state/outbox/<YYYY-MM-DD>_<topic>.txt` 에 저장
  2. listener가 매 폴 사이클 (≤30초) 끝에 outbox 체크 → 새 파일 발견 시 텔레그램 sendMessage 호출
  3. 송신 성공 시 `_state/outbox_sent/` 로 이동 (실패 시 outbox에 잔존, 다음 사이클 재시도)
  4. 4096자 초과는 3800자 청크 분할 + `[1/N]` 프리픽스
- **표준 메시지 포맷** (신규 수집 결과):
  ```
  [신규 수집 결과 YYYY-MM-DD]
  D-7 기준: YYYY-MM-DD / 다음 번호: NNN~

  [A] 라인 / 토픽 요약
    D-7: ✓/△/✗ / 매장정합: ◎/○ / 비고
  ... (라인별 후보)

  [정직한 한계]
  - 검증 실패·약한 신호 명시

  [권장 발행]
  NNN / line / 토픽

  [결정 옵션]
  1. 안전안 (권장)
  2. 확장안
  3. 직접 지정
  ```
- **공용 헬퍼**: `automation/scripts/tg_send.py`
  - 모듈로: `from tg_send import send_text; send_text("msg")`
  - CLI로: `py -3 automation/scripts/tg_send.py <txt_path>` — 1회 임시 송신용
- **listener 코드**: `check_outbox(trusted)` — `poll_loop()` 매 사이클 끝에 호출
- **트리거**: 사장님이 신규 수집 명령 (`신규 수집`, `news`, `scam`, `tip`, `qa`, 또는 채팅 직접) → 클로드는 결과 표를 outbox에 즉시 떨굼 → 자동 푸시
- **재시작 필요**: listener.py 수정 후 1회 재시작 (`tasklist | findstr python` 으로 PID 확인 → kill → `start_telegram_listener.bat`)

## Step 3: 이미지 프롬프트 (v6 — 카드별 독립 + 본문 직역 + 밝은 톤)

### 핵심 룰
① **본문 직역**: 각 카드 본문(headline+body)을 그대로 그림. 추상 은유 ❌. 판단 — "사진만 보고 카드 주제가 떠오르는가?"
② **각 카드 독립**: 5장 시리즈 통일 신경 X. 색감·소품·환경 다 달라도 OK.
③ **밝은 톤**: soft warm white #F5F0E8 / cream / pale beige 배경. 검정 배경·드라마틱 스포트라이트 ❌.
④ phonespot orange #F74B0B 액센트 소량. NO faces / NO logos / NO text on screens / 숫자·날짜 소품 흐림. Photorealistic editorial photography.

### 표준 프롬프트 양식 (★ 슬러그당 통합 블록 1개 = 5장 한번에 복사)
복사 버튼 **1번**으로 5장을 모두 입력할 수 있도록 **공통 룰 1회 + 카드별 장면 5개를 단일 블록에 통합**한다. 카드마다 별도 블록으로 쪼개지 않는다 (옛 룰 폐기).

블록 양식 (슬러그 1개당 1블록):
```

★ 각각 별개 이미지 파일로 생성 ★

공통 룰 (5장 모두 적용):
1080x1080, photorealistic editorial. Bright airy mood, light cream / warm white / pale beige background. No black background, no dramatic spotlight, no deep shadow. No text on screens, no brand logos, no real human faces. Numbers/dates blurred. Phonespot orange #F74B0B accent if subtle.
각 이미지는 위 룰을 따르되, 색감·소품·구도는 카드별 독립. "Same tone as image 1" 같은 시리즈 통일 지시 ❌.

— 1.png — [카드 1 헤드라인]
[장면 1~2문장: 누가 / 무엇이 / 어디서. 소품·손짓·배치 명시]

— 2.png — [카드 2 헤드라인]
[장면 1~2문장]

— 3.png — [카드 3 헤드라인]
[장면 1~2문장]

— 4.png — [카드 4 헤드라인]
[장면 1~2문장]

— 5.png — [카드 5 헤드라인]
[장면 1~2문장]
```

핵심 원칙:
- 공통 룰은 블록 상단에 **1회만** 기술 (5번 반복 ❌ → 토큰 절약 + 사용자 복사 1회)
- 카드별 장면 5개는 한 블록 내에서 `— N.png —` 헤더로 분리 명시
- "각 카드 독립" 원칙은 그대로: 시리즈 통일 톤 지시 금지
- 다중 발행 시 슬러그 N개 → 통합 블록 N개 (슬러그별로 1블록씩)

### 프롬프트 폴더 저장 (★ 2026-05-30 신설 — 채팅방 없이 재활용)
- JSON 작성과 **동시에** `cardnews/images/NNN_<slug>/prompt.md` 에 통합 프롬프트 1블록 Write
- 효과: 채팅방 안 들어가도 사장님이 이미지 폴더 안에서 프롬프트 바로 복사 가능 → 재생성·재시도·다른 PC 작업 자유
- 파일명 고정: `prompt.md` (확장자 .md). cardnews_renderer가 카드 이미지로 오인 X (코드는 png/jpg/jpeg/webp만 후보로 봄)
- 양식: 위 「표준 프롬프트 양식」 통째로 (공통 룰 + 5장 각각의 장면)

### 본문 ↔ 직역 장면 매칭
- 특정 제품 단독 → 그 폰 단독 히어로샷, 밝은 크림 배경
- 마감 임박 → 달력·플래너 + 마감일 마커, 폰 옆에
- 금전 보상 → 손짓 + 봉투/지폐 가장자리 (숫자 불가독)

- 라인업 → top-down flat lay, 여러 기기 정렬

## Step 4: 콘텐츠 작성 (articles/<slug>.json 단일 파일)

### 슬러그 명명 규칙 (★ 3자리 번호 + 콘텐츠 라인 type — 2026-05-30 갱신)
- **형식**: `NNN_<type>_<topic_slug>` (번호 3자리 + 콘텐츠 라인 + 토픽)
  - 예: `003_news_galaxy_zfold8`, `004_scam_voice_phishing_q2`, `005_tip_battery_save`, `006_qa_seonggi_vs_branch`
- **type**: `news` / `scam` / `tip` / `qa` (4 라인 MVP — 자세한 라인 정의는 아래 별도 섹션)
- **다음 번호 계산** (★ 매 신규 작성 전 반드시 1회):
  1. `cardnews/articles/*.json` 스캔
  2. 파일명이 `^(\d{3})_` 패턴 매칭하는 것만 추출
  3. 추출된 번호 중 **최대값 + 1** 이 다음 신규 번호
  4. 매칭 0건이면 `001`부터
  5. 다중 발행 N건이면 연속 N개 번호 부여 (예: 직전 max=019 → 020, 021, 022)
- **적용 범위** (3곳 모두 동일 prefix):
  - `cardnews/articles/NNN_slug.json`
  - `cardnews/images/NNN_slug/`
  - `cardnews/output/NNN_slug/` (렌더링 시 자동 생성)
- **JSON 내부 `slug` 필드도 동일하게** `NNN_<topic_slug>` 로 기록
- **기존 19개 슬러그**: 2026-05-29 이전 생성분은 prefix 없이 유지 (소급 적용 ❌)
- **내부 호출**: 사장님–AI 소통 시 "1번", "5번" 등 번호로 슬러그 식별 가능

### 필드 구성
필드: `slug`, `content_type`, `title`, `publication_date`, `source_line`, `cards`(6장), `captions_md`(5채널 통째), `narration_md`(영상용 풀 나레이션).
- `content_type` 신설: `"news"` / `"scam"` / `"tip"` / `"qa"` 중 하나. 슬러그 prefix와 일치 필수

### narration_md 필드 (★ 영상 task 기준 지표)
- **목적**: 카드뉴스를 60초 뉴스 방송으로 만들 때 길이·페이싱·톤의 기준선. 영상 task가 청크 분할·TTS 페이싱 기준으로 사용
- **분량**: 한글 **500~700자** (edge-tts RATE +50% × 60초 ≈ 540~600자 분량)
- **톤**: 뉴스 앵커. `~합니다 / ~입니다` 어미. 카드 본문의 구어체보다 정제. 신뢰감·간결·중립
- **구조** (마크다운 헤더 명시):
  ```
  ## 인트로
  [훅 1~2문장 — 시청자 주의 환기 + 핵심 한 줄]
  
  ## 본문
  [4~5문단. 카드 1~5 내용을 흐름으로 풀어쓰기. 카드 = 한 문단 ≠ 강제.]
  
  ## 매장 안내
  [폰스팟 광교점 1~2문장 — 시청자 행동 유도]
  ```
- **줄바꿈 규칙**: **문장 단위 줄바꿈** (영상 task가 caption_chunks로 자를 때 자연 분할선). 한 문단 안에서도 문장 끝마다 줄바꿈.
- **금지**:
  - "안녕하세요", "오늘은" 같은 블로그·인스타 인사말 (뉴스 방송이라서 X)
  - `{LITTLY}` `{PRECON_URL}` 토큰 (이건 captions_md 전용. 나레이션 음성엔 URL 안 읽음)
  - 해시태그·이모지
  - 카드 본문의 `<span class="hl">` HTML 태그 (텍스트만)

### 권장 본문 흐름 (방송 톤 가이드)
1. 인트로: "오늘의 IT 소식입니다." / "[키워드]에 새로운 변화가 있습니다." 류
2. 본문 첫 문단: 5W 핵심 (누가·무엇·언제 보도)
3. 본문 중간: 어떻게·왜 (메커니즘·차별점)
4. 본문 끝: 시청자에게 의미 (개인정보·비용·편의)
5. 매장 안내: "더 자세한 안내는 폰스팟 광교점에서 받으실 수 있습니다."
각 카드 = `image`, `headline`, `body`, `source`.
- 헤드라인 명사형, `\n` 줄바꿈, `<span class="hl">강조</span>`로 오렌지 포인트
- 본문 1~2문장 서술어 완성. "원UI" → "ONE UI" 표기
- card_6은 폰스팟 매장 안내 고정
- 캡션 내 `{LITTLY}` `{PRECON_URL}` 토큰 사용 (renderer 자동 치환)
- 최하단 `[사전승낙서] {PRECON_URL}` 필수

- **CTA**: litt.ly/phonespot 1줄. 매장 정보 풀세트 X

# 후킹 룰 (★ 2026-05-29 신설 — 클릭·정독·전환 최적화)

## 0. 원칙 (★ 매번 적용)
- **후킹 ≠ 클릭베이트**. 매장 신뢰도 손상하는 과장·낚시·거짓 헤드라인 ❌
- **후킹 = 본문에 실제 있는 가장 강한 정보를 앞에 노출**하는 기술
- 폰스팟은 매장 비즈니스 = **신뢰가 자산**. 거짓 클릭은 즉시 매장 손해로 환산됨
- 카드 1 헤드라인 + 캡션 첫 줄 = **두 번의 후킹**. 둘은 같은 정보의 다른 표현

## 1. 후킹 6타입 상세

### 숫자형 (Numbers)
- **공식**: `[숫자/통계] + 핵심 + 행동`
- **예**: "아이폰 사용자 87%가 모르는 자동 잠금 신기능"
- **적합 토픽**: 신기능·통계·비교·시장점유
- **강도**: 강 / **신뢰도**: ★★★★★ (출처만 명시되면)
- **주의**: 숫자 정확. 본문에 출처 (매체+발행일) 명시

### 의문형 (Question)
- **공식**: `[궁금증] ~ 정말 그럴까?` 또는 `~ 어떻게 될까?`
- **예**: "도난당한 아이폰, 정말 자동으로 잠긴다?"
- **적합 토픽**: 신기능·루머·전망·소비자 궁금증
- **강도**: 중 / **신뢰도**: ★★★★
- **주의**: 본문에서 **반드시 답을 줘야**. Curiosity gap 채움 의무

### 손실회피형 (Loss aversion)
- **공식**: `이거 모르면 / 안 하면 ~ 손해`
- **예**: "지금 도난 기기 보호 안 켜두면, 도난 시 비밀번호까지 털린다"
- **적합 토픽**: 보안·꿀팁·놓치기 쉬운 설정·시즌 행사
- **강도**: 강 / **신뢰도**: ★★★
- **주의**: 과한 공포 마케팅 X (절제). 본문에서 해결책 제시

### 충격형 (Shock)
- **공식**: `[놀라운 사실] / [반전]`
- **예**: "5초 안에 아이폰이 잠긴다 — 가속도계만으로"
- **적합 토픽**: 신기술·반전 정보·통념 깨기
- **강도**: 강 / **신뢰도**: ★★★
- **주의**: 검증된 팩트만. 추측·루머 단정 ❌

### 시간압박형 (Urgency)
- **공식**: `[D-N / 마감 / 곧] + 행동`
- **예**: "WWDC26 D-10 / 6월 8일 발표 임박"
- **적합 토픽**: 출시·이벤트·마감·예약
- **강도**: 강 / **신뢰도**: ★★★★★
- **주의**: 가짜 마감·허위 D-N ❌. 실제 일정만

### 친근 vs 신뢰체 (Tone)
- **친근체**: `~잖아요`, `~네요`, `~ㅂ니다` 부드럽게 — 인스타·스레드·틱톡
- **신뢰체**: `~합니다`, `~입니다`, `~ㄴ다` — 블로그·유튜브
- **카드 1 헤드라인은 모든 채널 공통**이라 명사형 + 신뢰 톤 유지
- **캡션 첫 줄에서만** 채널별 톤 변형

## 2. 토픽 → 후킹 자동 매핑 (★ 캡션 작성 시 자동 추론)
