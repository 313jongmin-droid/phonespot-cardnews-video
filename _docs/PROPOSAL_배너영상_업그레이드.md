# 영상 트랙 아키텍처 + 배너 추가 — 마스터 기획서 (2026-06-19)

> **기획 전부 확정해두고 구현은 1개씩**(종민). 구현 완료분은 SYSTEM_MAP C단원 가이드로 승격.

---

## 0. 분류(taxonomy) — "타이포광고/배너광고" ❌ → **영상 = 타입 × 용도**

기존 "타이포광고·배너광고"는 제작방식과 용도가 섞인 이름. 깔끔하게 **2축 분리**:

**축1 — 생성 타입(파이프라인, = 패널 탭)**
| 타입 | 입력 → 비주얼 | 오디오 | 상태 |
|---|---|---|---|
| **카드뉴스** | 기사 → 일러/실사 매칭 | TTS + 정밀싱크 자막 | 운영중 |
| **타이포** | MD 스크립트 → 모션그래픽 23스타일 | 무나레이션 = SFX+음악 | 구현됨(promo) |
| **배너** | 배너 이미지 N장 → 풀블리드 슬라이드 | TTS + 선택 BGM | **신규(이번)** |
| **실사AI** | 프롬프트 → Higgsfield 영상 → ffmpeg | (트랙별) | 구현됨(promo_ai) |

**축2 — 용도(= 퍼블리시/캡션 분기)**
- **콘텐츠(오가닉)**: 5채널 캡션(네이버/스레드/인스타/유튜브/틱톡) + 사전승낙서·링크 → 주로 카드뉴스
- **광고**: 광고 전용 캡션(헤드라인/본문/CTA) → 메타·당근 → 타이포·배너·실사AI

> **왜 이게 더 나은가:** ① 타입명이 *제작방식*을 정확히 가리킴(타이포=모션, 배너=배너슬라이드) ② 용도(콘텐츠/광고)는 *속성*이라 같은 타입도 둘 다 가능(예: 카드뉴스를 광고로도) ③ 빠졌던 **실사AI**까지 한 체계에 들어감 ④ 패널 탭 = 타입 4개로 단순, 용도는 렌더 끝 publish에서 갈림.

**패널 트랙 탭 = 카드뉴스 / 타이포 / 배너 / 실사AI** (4개). 각 영상에 용도 라벨(콘텐츠/광고)이 붙어 캡션 빌더를 가름.

## 1. 확정 사항 (종민 결정)

| 항목 | 결정 |
|---|---|
| 방식 | **A** — 카드뉴스 Remotion 엔진 재사용 + "배너 모드"(별도 컴포지션) |
| 기존 보존 | **★ 카드뉴스·타이포·실사AI 트랙 100% 무수정**. 배너는 비파괴 추가 |
| 배너 출처 | 프롬프트(생성기) → GPT/캔바 제작 → 드롭폴더 → 패널 import (현 카드이미지 흐름 동일) |
| 자막 | **on/off 선택**(배너에 텍스트 있으니 기본 OFF, 켜면 TTS 정밀싱크 재사용) |
| 규격 | **9:16만 시작** + 코드 width/height 파라미터화 → 미래 1:1·4:5 무리워크 |
| 퍼블리시 | **광고 전용 캡션**(메타·당근, AD_COPY.txt). 사전승낙서·링크 토큰 재사용 |
| 오디오 | TTS(필수) + BGM 음악풀(선택) |
| 분량 | 배너 3~5장 · 15~20초 |

---

## 3. 신규 컴포지션 `BannerAdShort` (9:16, 비율 파라미터)
- 섹션별: 풀블리드 배너 + 켄번스 + TTS(배너별 1줄) + (옵션)정밀싱크 자막 + 마지막 CTA(휴대폰성지 폰스팟 토큰).
- 헤더/카드캡션 없음. 자막 기본 OFF.
- `width/height` 파라미터 → 지금 1080×1920만 등록, 컴포넌트는 비율 적응 작성(미래 1:1·4:5 등록만 추가).

## 4. `build_banner.py` (신규)
- 배너 N장 + 배너별 TTS 1줄 + 자막여부 + BGM선택 → `output/<slug>/banner_script.json`. casual `build_script` 무수정.

## 5. 광고 캡션 빌더 (신규)
- 광고 헤드라인+본문+CTA+사전승낙서+litt.ly → `RESULTS/<slug>/AD_COPY.txt`. 5채널 오가닉 아님.

## 6. 패널 UI — 타입 탭

```
┌──────────────────────────────────────────────┐
│ [카드뉴스] [타이포] [배너] [실사AI]          │ ← 생성 타입 탭 (localStorage 기억)
├──────────────────────────────────────────────┤
│  카드뉴스 = 현행 화면 그대로 (무변경)          │
│  타이포  = promo 목록·렌더                    │
│  배너   = import·대본·자막토글·렌더 (신규)    │
│  실사AI = promo_ai 워크플로                    │
│  (각 영상에 용도 라벨: 콘텐츠 / 광고)          │
├──────────────────────────────────────────────┤
│  ▾ 공통: 라이브러리 · 시스템 관리 (접이식)     │
└──────────────────────────────────────────────┘
```
- `server.py INDEX_HTML` 탭 JS + 섹션 display 토글. **165KB = bash-python(I단원)** + `PANEL_VERSION`↑ + 개폐 검증.

## 7. 전체 확장 기획 (지금 다 설계, 구현은 나중 1개씩)

| # | 확장 | 설계 | 의존 |
|---|---|---|---|
| E1 | **규격 1:1·4:5** | `BannerAdShort`에 비율 등록 추가 + CTA/자막 비율 적응 + 패널 드롭다운 활성화 | 배너 트랙 |
| E2 | **광고 생성기 자동 연동** | `generator.html`이 배너 프롬프트/이미지를 드롭폴더로 export → 패널 자동 감지·import | 배너 트랙 + 광고 task |
| E3 | **배너↔TTS 자동매칭** | 배너 순서 또는 의미매칭(CLIP)으로 어느 배너에 어느 대사 자동 배정 | 배너 트랙 + E단원 |
| E4 | **실사AI 탭 통합** | 현재 별도 promo_ai → 패널 타입 탭으로 흡수(워크플로 노출) | 패널 탭 |
| E5 | **타입 간 변환** | 카드뉴스→배너, 배너→타이포 등 소재 재활용 | 전체 |

## 8. 구현 로드맵 (1개씩)

```
[현재] 카드뉴스 · 타이포 · 실사AI 트랙 운영중
  │
  ├─ STEP 1: BannerAdShort 컴포지션 (9:16, 비율 파라미터)
  ├─ STEP 2: build_banner.py + run_banner.bat
  ├─ STEP 3: 광고 캡션 빌더 (AD_COPY.txt)
  ├─ STEP 4: 패널 banner_ad_* 액션 (import·대본·자막토글·렌더)
  ├─ STEP 5: 패널 타입 탭 UI 재개편 (카드뉴스 화면 보존)
  ├─ STEP 6: 검증 — 배너 3장 테스트 렌더(렌더PC)
  │
  └─ [확장] E1 규격 → E2 생성기연동 → E3 자동매칭 → E4 실사AI탭 → E5 변환 (필요 순서로)
```
- 각 STEP/E 항목 = **독립 비파괴**. 하나 끝낼 때마다 검증 + 가이드 박기 + push(cmd).
- 원칙: casual/타이포/실사AI 무수정, 하나씩, 검증 후 다음.

→ **타입 분류(카드뉴스/타이포/배너/실사AI) OK면 STEP 1부터 시작.** 분류 더 손볼 거면 알려줘.
