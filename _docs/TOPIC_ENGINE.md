# 주제 생성기 (TOPIC ENGINE) — SNS 떡상 주제 발굴 정본

> **격상 (2026-06-23, 종민)**: 기존 "뉴스 RSS 수집기"를 **"SNS(유튜브·인스타) 떡상용 주제 생성기"로 격상.**
> 출력 트랙이 카드뉴스 1개 → 카드뉴스영상 + 실사 viral까지 넓어졌으므로, 주제는 더 이상 "뉴스 기사"에 갇히지 않는다.
> **클로드 역할 = 주제 생성(이 문서). 구현(이미지·카드 렌더·실사 영상)은 별개 트랙.** 주제↔구현 분리.

---

## 0. 역할 경계 (제일 중요)

| 단계 | 담당 | 산출 |
|---|---|---|
| **① 주제·소재 발굴 + 후킹 슬로건** | **클로드 (이 문서)** | 떡상 주제 풀 — **발굴 + 후킹 슬로건/스크립트 방향이 본질**. 트랙·포맷은 참고 태그 |
| ② 구현 — 카드뉴스 | 카드 빌드(`CARDNEWS_BUILD`) → 부사수 PC `run_pngs.bat` | 18 JPG |
| ② 구현 — 카드뉴스영상 | shorts(`INSTRUCTIONS_SHORTS`, Typecast+MoviePy) | mp4 (무료) |
| ② 구현 — 실사 viral | promo_ai(`shorts/promo_ai`, Higgsfield Kling 3.0) | mp4 (크레딧) |

클로드는 ①만 한다. ②는 주제를 받아 각 트랙이 제작. **주제 생성 시 "이건 어느 트랙으로 가면 떡상한다"를 함께 제시**하되, 실제 제작은 분리.

**★ 산출 영속화 = `cardnews/topics/topic_pool.json`** (git 추적). 발굴 주제를 여기 적재 → 카드·카드뉴스영상·실사 task가 **pull해서 `status:"candidate"` 중 골라 제작**(채팅 안 봐도 됨). 발행되면 `status:"published"` + `slug` 기록. 미발행 후보는 carryover(`content_guide §3.5`, _state·gitignore=로컬용)와 달리 **git으로 멀티PC 전파**된다. 매 "주제 생성" 끝에 신규/변경분을 이 파일에 반영.

**★ 패널 연동 (webui `/topics`, 2026-06-23)**: 패널에서 주제 풀을 표로 보고 행마다 `[카드][카드영상][실사]` 배정 → `status:"assigned"` + `assigned_track` 기록(app.py `topics_assign`). slug 있는 항목은 "렌더 →"로 기존 render-stream 트리거. **배정·렌더는 로컬 PC 패널에서만(push 권한), 부사수 PC는 git pull로 받아 제작**(STEP 0 멀티PC 규칙). 각 항목 `brief`(=summary 약식 골격, 배정 시 클로드가 track별 상세 확장)로 task가 뼈대까지 받음.

## 1. 입력 소스 5갈래 (기존 ①만 → ①~⑤로 확장)

| # | 소스 | 무엇을 주나 | 비고 |
|---|---|---|---|
| ① | **뉴스·트렌드** RSS(`_state/news_feed.json`) + **NaverSearch(★ `search_news` date순 + `datalab_search` 검색수요 + `search_blog`)** + WebSearch(보조) | 시의성(D-7) + **검색수요 우선순위** | ★네이버가 한국 매체·트렌드 커버리지 최고. **datalab로 소재별 검색량 비교 → 저수요 소재 걸러냄**(예: 통신비결합 ≪ 아이폰18). RSS는 패널 켤 때 자동 갱신 |
| ② | **성과 4채널** 유튜브_인사이트 + **인스타·틱톡·스레드 시트** | 떡상 패턴(Top 키워드·후킹). ★**채널별 잘 먹히는 소재가 다름**: 인스타 릴스=호구·가짜성지 최고(가짜성지 208), 틱톡=유심(LG유플 34,300)·S26·애플페이·에어팟 강세(가짜성지는 틱톡선 129로 낮음), 유튜브=애플·유심, 스레드=일상·시세 소통형. 공통 강세=애플·유심·S26·애플페이 | **매 생성 시 4채널 다 Read 필수**(유튜브·인스타만 보고 틱톡·스레드 누락 ❌ — 2026-07-14 확인). 클로드 자율 가중, 채널별 소재 반응 구분 |
| ③ | **트렌드·밈** WebSearch(밈·챌린지·공감 소재) | 바이럴 각도(meme→실사 viral) | ★신규 편입 |
| ④ | **시즌·라이프** (달력·생활 맥락) | 휴가·명절·개학 등 시점 소재 | life/tip |
| ⑤ | **carryover** (`content_guide.md §3.5`) | 미선택 검증 후보 재등판 | 발행 시 삭제, 시점만료 제거 |

사장님 URL 주면 WebFetch 최우선(②③ 위에). 회피키워드(통합요금제·요금인하·알뜰폰 가성비·MVNO·자급제+저가요금제)는 매장 정합 위배 → 제외.

## 2. 떡상점수 (정량 라벨 — 주제 우선순위)

성과시트 가중 룰을 그대로 산식화한다.

```
떡상점수 = 성과 키워드 매치(+30%)        # 유튜브_인사이트 Top 키워드 포함
         + 후킹 패턴 매치(+20%)          # 의문문·숫자·반전 등 우수 후킹
         + 시의성/트렌드(+)              # news D-7 이내 / 트렌드 상승 / 시즌 적중
         + 매장 정합(+)                  # 고가단말·지원금·통신 도메인
         − 회피키워드 / dup (제외)
```

- **apple+ 가중 (2026-06-22 관찰)**: 애플 토픽(아이폰·iOS·애플페이·에어팟·애플워치)이 갤럭시보다 조회 우위 경향 → 가산. 표본 누적되며 재검증.
- 라벨 예: `apple+·검증·D-7✅·yt매치` / `트렌드·후킹강` 등. 점수는 상대 순위용(절대수치 X).

## 3. 출력 스키마 (주제 풀 표)

매 "주제 생성" 시 아래 표로 제출. **검증 완료 주제만**(미검증·"사실확인 필요" 등재 ❌).

**클로드 산출의 무게중심 = ① 주제/소재 발굴 ② 후킹 슬로건(스크립트 방향까지).** 떡상은 "뭘 다루냐 + 첫 1초에 뭐라 거느냐"에서 갈린다. 트랙·포맷은 **참고 태그**(부차, 구현팀이 조정 가능).

| 컬럼 | 내용 | 비중 |
|---|---|---|
| **주제/소재** | 발굴한 떡상 주제 (뉴스 아니어도 됨) | ★핵심 |
| **후킹 슬로건** | 첫 1초·제목 후크 1~2개. **`cardnews/topics/hook_formulas.md` 19공식에서 주제·라인에 맞는 1~2개 선택** + 유튜브_인사이트 실측 패턴(의문·숫자·반전·시간·가격) 결합 | ★핵심 |
| **한 줄 요약** | 스크립트 방향(핵심 메시지 + 카드/컷 흐름) | ★핵심 |
| 떡상점수 | 성과키워드·후킹·시의성·매장정합 라벨(§2) + 검증 출처 | 핵심 |
| 라인·추천트랙·포맷 | news~life / 카드·카드영상·실사 viral / 쇼츠 포맷 | 참고 태그 |

날짜는 **env 'Today's date'만** 사용(STEP 0 / PREFLIGHT 날짜 규칙). 첫 줄에 `오늘(env)·D-7` 복창.

**★ 채팅 출력 표기 규칙 (2026-06-23 종민 확정)**:
1. **번호 = 1~N 정상 숫자.** `N1`·`C01` 등 라인 접두어 ❌ (라인은 태그 컬럼에 있음).
2. **hook 뒤에 `[공식]` 라벨 병기** — `hook_formulas.md` 번호로. 예: `"…30만원 더 비싸?" [⑯질문+⑥비교결론]`.
3. **컬럼 순서(고정, 2026-07-21 확정)** = `# | 주제 | 🎣 후킹 [공식] | 채널 | 점수 | 실사AI`. **7라인별 섹션으로 묶어** 출력(📰news 🚨scam 💡tip ❓qa 🆚pick 😆meme 🌿life), 섹션 제목 옆에 주력 채널 아이콘 병기. 한 줄 요약은 표에서 빼고 `topic_pool.summary`로만 관리(표 폭 절약). **채널 아이콘**: 📸인스타 🎬틱톡 ▶️유튜브 📝블로그 🧵스레드.

   **출력 템플릿 (이 형태를 그대로 유지 — 컬럼 추가·삭제 임의 변경 ❌)**
   ```
   **📰 news** (🎬▶️)
   | # | 주제 | 🎣 후킹 [공식] | 채널 | 점수 | 실사AI |
   |---|---|---|---|---|---|
   | 1 | 폰 개통 안면인증 의무화 | "폰 개통할 때 얼굴 인증?" `[②+⑯]` | 🎬📸▶️ | ★검증·D-7 | 추천 |
   … (7라인 각 3개 이상, 라인별 섹션 반복)

   **실사각색 (추천·조건부만)**
   - 1 → `실사각색: "폰 개통하러 갔는데 얼굴을 찍으래" → 알고 보니 8월부터 전 매장 의무`
   ```
4. topic_pool 내부 `id`(035·C01…)는 **파일 식별용** — 채팅 표엔 노출하지 않는다(1~N 번호로만).
5. **★ 실사AI 적합도 태그 필수 (2026-07-21 종민)**: **기존 표 형식(번호│주제│후킹[공식]│채널│점수)을 그대로 두고 `실사AI` 컬럼 1개만 추가**한다(값 = 추천/조건부/비추천). 표 구조를 바꾸지 말 것. **각색 힌트는 표 아래에 추천·조건부 항목만 `실사각색: <후킹 1문장> → <반전 1문장>` 1줄씩 모아 표기**. 판별 정본 = `shorts/promo_ai/LIVE_ACTION_FIT.md`(소유=promo_ai task, 주제 task는 참조만).
   - **필수 3조건**(하나라도 ❌=비추천): ①1인칭 썰 재구성 가능(문제→빌드업→반전) ②사람 표정·리액션으로 표현 가능(분노·공감·손실·반전) ③3~6컷 15~25초 압축 가능
   - **가점**: 기존 컷 재활용 +2 / 누구나 들어본 대사·멘트 기반 +2 / 정찰제·실구매가 CTA 직결 +1 / in-image 텍스트 불필요 +1
   - **감점**: 체크리스트·절차형 -3 / 표·그래프가 본질 -3 / 법령·수치 설명 위주 -2 / 신규 장소·소품 필요 -2
   - **판정**: 5점↑ 추천 / 2~4점 조건부 / 1점↓ 비추천. 기준선 = "사람이 겪은 일 + 감정이 움직임 + 3~6컷"=실사 / "확인할 것 N가지 + 표"=카드

**★ 발굴 분량 규칙 (2026-07-09 → 07-22 종민 갱신)**: 매 주제 수집 시 **scam 제외 6라인**(news·tip·qa·pick·meme·life), **news는 6개(2배)** + tip·qa·pick·meme·life 각 3개 = **21개 이상**. scam은 발굴 목록에서 뺀다(2026-07-22 종민). 못 채우면 그 라인만 사유 1줄. 표는 라인별로 묶어 제시.
> **★ news 필수 포함 (2026-07-22 종민 지적 — 자주 놓침)**: **통신사 요금제·결합·멤버십·지원금 정책 변경(종료·축소·개편·신설)**을 news에 반드시 포함한다. 예: SKT 온가족할인 7/31 종료, 통신3사 결합할인 축소, 통합요금제 QoS. **매장정합 최강**(소비자 손해→매장 상담 기회)인데 제품 뉴스만 보다 놓치기 쉬움. 네이버 `search_news "통신사 결합 종료/개편"`을 news 수집 시 상시 1회 돌릴 것.

## 4. 라인 → 추천 트랙 매핑 (주제 성격이 트랙을 정한다)

| 라인 | 성격 | 추천 트랙 | 이유 |
|---|---|---|---|
| news | 속보·수치 | 카드 / 카드뉴스영상 | 출처·날짜·수치 정확 |
| scam | 경고·단계 | 카드 / 카드뉴스영상 | 수법→대응 단계 |
| qa | 비교·계산 | 카드 / 카드뉴스영상 | 표·수치 가독성 |
| tip | 저장·리스트 | 카드 / 카드뉴스영상 | TOP5 리스트형 |
| pick | 스펙 비교 | 카드 / 카드뉴스영상 | 비교표 가독성 (실사 부적합) — 확정 2026-06-23 |
| **meme** | 반전·풍자 | **실사 viral (하이브리드)** | 후킹·체류·알고리즘. **실사 + 핵심 수치 자막 병행** — 확정 2026-06-23 |
| life | 상황·경고 | 카드 / 카드뉴스영상 | 세팅법·절차 정보 중심 — 확정 2026-06-23 (상황극 한정 실사 예외) |

> **meme→실사 구현 정본 = `shorts/promo_ai/MEME_TO_VIRAL.md`** (7비트 매핑·8단계·안전·비용). 전략·리스크·파일럿 = `기획안_meme_실사영상_분담`.
> **타이포(promo) = 1급 선택 트랙 (2026-06-24 라우팅 추가).** 어느 라인이든 **슬로건·정책·신뢰·데이터 강조** 각도면 타이포로 구현 가능. 특히 qa/pick/tip의 수치·비교, life의 정책·절차, 그리고 **브랜드 가치 주제(정찰제·호갱방지·가격투명류)는 타이포 1순위.** 주제 seed → promo MD 변환 정본 = `shorts/promo/TOPIC_TO_PROMO.md`.
> **실사 ad** = 매장·실사 광고 컷(`shorts/promo_ai`의 ad 카테고리). meme=viral, ad=광고.
> **트랙 5종 = 카드뉴스 / 카드뉴스영상 / 타이포 / 실사 viral / 실사 ad.** 주제 풀은 단일, **번호+트랙 지정으로 분배(§6).** 추천트랙은 참고 태그일 뿐 — 사장님이 같은 주제를 원하는 스타일로 선택.

## 5. 검증·중복 규칙 (게이트)

1. **사실 검증 완료만 등재** — 특정 사건/뉴스는 WebSearch·WebFetch로 실체·시점·매장정합 확인. 부적합이면 제외 근거 1줄.
2. **news = D-7 strict** — 보도일 기준 `news_d7_filter.py`. meme/tip/qa/pick/life는 시점 무관.
3. **dup 회피** — `cardnews/articles/*.json` + `content_guide.md §2` 교차. 발행분 재제안 ❌.
4. **회피키워드** — §1 회피 목록 제외.
5. 미선택 검증 후보 → `content_guide.md §3.5` carryover 적재(다음 생성 시 재등판).
