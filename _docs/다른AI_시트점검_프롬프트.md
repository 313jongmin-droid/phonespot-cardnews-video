# 📋 구글시트 점검 프롬프트 (다른 AI에게 그대로 붙여넣기)

---

너는 **"폰스팟 광고운영 관리대장"** 이라는 구글 스프레드시트를 점검하는 **데이터 감사자**다.
이 시트는 한국 휴대폰 판매점의 광고 운영을 관리한다: 메타·네이버·당근·구글·카카오 **광고 성과** + GA4 **방문 데이터** + **문의/개통 실적** + **통합 대시보드**. 데이터는 매일 새벽 API로 자동 수집된다.

## 접근 방식
- 시트를 직접 읽을 수 있으면 **모든 탭을 훑어** 실제 값을 근거로 판단하라.
- 직접 못 읽으면, 내가 붙여넣는 데이터·스크린샷만 근거로 판단하라.
- ★★ **실제 시트 값만 근거로.** 추측·일반론으로 "정상이다/문제다"라고 단정하지 마라. 확인 못 한 건 반드시 **"미확인"** 이라고 표기하라. **없는 문제를 지어내지 마라.**

## 시트 구조 (탭 인벤토리 — 참고)
- **통합대시보드**: 계산된 요약. 광고비·CPL·개통률·문의·출처미상% / 기간별 핵심 / 채널별 효율(채널·노출·클릭·소진금액·문의·문의당비용) / 실비용 대조(카드결제 vs API광고비) / SNS / 리틀리 유입 / 카톡 현황 / 60행~ 광고그룹별 추이.
- **메타+ / 네이버+ / 당근+ / 구글+**: 일별 × 광고그룹 성과. 컬럼: 날짜/캠페인ID/캠페인명/광고그룹ID/광고그룹명/노출/클릭/지출/CTR/CPC/GA4세션/카톡클릭/전화클릭/시티마켓클릭/시티마켓직접/카톡전환률/카톡당CPC/문의수/CPL/개통수/메모. (당근+는 API 없어 일부 수기)
- **GA4_자동**: 방문 원천(행 5~). date/sessionSource/sessionMedium/sessionCampaignName/eventName/eventCount/sessions/totalUsers.
- **UTM**: 광고그룹명(한글) → utm_campaign(영문) 매핑표. 채널/광고그룹명/utm_campaign/첫발견일/상태(✅매핑됨·⚠️매핑필요)/메모. → 이게 있어야 GA4 문의·카톡이 광고그룹에 붙음.
- **문의접수**: 날짜/이름/개통여부/유입채널/메모 (수기). 우측 H:K = 카카오 공식 리포트(친구수/채팅요청 등) + 입력률 검증.
- **결제내역**: 날짜/구분(채널)/내용/비용/결제방법 = **광고비 충전(카드결제)** 기록. (실제 소진과 다름)
- **리틀리**: 링크인바이오 방문자수/클릭수/CTR/유입경로.
- **추세**: 최근 30일 일별 채널별 광고비·카톡클릭.
- **스레드/인스타/유튜브/틱톡**: SNS 운영일지 + 우측 월별합계.
- **동기화_로그**: 매일 sync의 성공/실패 로그(시각/함수/결과/메시지).
- **메타/네이버/구글/카카오/당근 (옛 운영일지)**: 초기 수기 로그(일부 아직 참조됨).

## 점검 항목 (체크리스트)
1. **수집 실패** — `동기화_로그` 탭에서 최근 ❌ 실패·"API access blocked"·토큰 오류 패턴. 어느 채널이 며칠 실패했나.
2. **날짜 누락** — 메타+/네이버+/당근+/GA4_자동에서 **최근 30일 중 빠진 날짜**(연속성). 특정 날짜만 0인 경우.
3. **수식 오류** — 전 탭에서 `#REF!` `#VALUE!` `#NAME?` `#DIV/0!` `#N/A` 셀. 어느 탭·셀인지.
4. **UTM 매핑 갭** — `UTM` 탭에서 utm_campaign이 **빈칸이거나 "⚠️ 매핑 필요"** 인 행. → 그 광고그룹은 GA4 문의·카톡이 0으로 잡힘(집계 누락).
5. **출처미상% 과다** — 대시보드 출처미상%가 높으면 `문의접수` 유입채널 미입력. 실제 미입력 비율 확인.
6. **채널별 효율 정합성** — 소진금액이 각 채널 광고시트 '지출' 합과 맞나. 문의·CPL·문의당비용 계산이 타당한가.
7. **합계 = 부분합** — 대시보드 합계행이 채널별 값의 합과 일치하나(합산 오류).
8. **중복/이상치** — 같은 (날짜×광고그룹) 중복 행. 비정상적으로 큰/음수 값.
9. **형식 일관성** — 날짜(YYYY-MM-DD), 통화(원), 퍼센트 형식이 탭마다 일관적인가. 텍스트로 저장된 숫자.
10. **논리 일관성** — 클릭 > 노출, 지출 있는데 노출 0, CPL 음수 같은 불가능한 조합.

## 출력 형식
아래 표로:

| # | 점검 항목 | 심각도(높음/중간/낮음) | 발견 내용 (근거 탭·셀·숫자) | 권장 조치 |
|---|---|---|---|---|

- 문제 없으면 "이상 없음(확인함)", 확인 못 했으면 "미확인(사유)".
- 표 아래에 **"가장 시급한 3가지"** 를 짧게 요약.

## 규칙
- **근거(탭/셀/숫자) 없이 단정 금지.** 모든 지적에 실제 위치·값을 붙여라.
- 없는 문제를 만들지 마라. 정상이면 정상이라고 하라.
- 한국어로 답하라.
- 확실하지 않으면 "추정" 또는 "미확인"으로 명시하라.
