# 광고관리 시트 수정 — 붙여넣기용 패치 (2026-06-08)

> 적용 방식: **라이브 Apps Script 에디터에 직접 붙여넣기** (시트 → `확장 프로그램` → `Apps Script`).
> repo 백업 `Code.gs`는 이미 같은 내용으로 갱신해 둠. 아래는 **라이브에 붙여넣을 조각**.
>
> ⚠️ 사실 확인된 전제: 라이브 `onOpen`은 백업과 다름(메타 메뉴 호출이 라이브엔 이미 있음). 그래서 onOpen은 **전체 교체 금지**, 한 줄만 추가.

---

## 항목 3 — 🎬 YouTube 메뉴 버튼 안 뜨는 문제 (가장 간단)

**원인(확정):** `youtube_sync.gs`의 `addYouTubeMenuItem()` 함수는 있는데, `onOpen()`에서 호출이 안 됨 → 메뉴가 안 생김.

**조치:** 라이브 `onOpen()` 함수의 **맨 끝 `}` 바로 앞**에 이 한 줄 추가:

```javascript
  try { addYouTubeMenuItem(); } catch (e) {}
```

- 시트를 새로고침(F5)하면 상단에 `🎬 YouTube` 메뉴가 뜸.
- 만약 `📡 메타 자동화` 메뉴도 안 보이면, 그 경우에만 아래 줄도 같이 추가:
  ```javascript
  try { buildMetaSyncMenu_(SpreadsheetApp.getUi()); } catch (e) {}
  ```
  (이미 메타 메뉴가 보이면 **추가하지 말 것** — 중복 메뉴 생김.)

## 항목 1 — 전체 업데이트 시 최근 업데이트 날짜/시간 기록

**조치 A:** 아래 함수를 `Code.gs`에 **새로 추가**(아무 함수 사이에 붙여넣기):

```javascript
// 최근 전체 업데이트 시각 기록 (통합대시보드 A46) — refreshAll이 호출
function recordLastRefresh_() {
  const stamp = Utilities.formatDate(new Date(), 'Asia/Seoul', 'yyyy-MM-dd HH:mm:ss');
  const sh = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('통합대시보드');
  if (sh) {
    sh.getRange('A46:F46').breakApart();
    sh.getRange('A46').setValue('🕐 마지막 전체 업데이트: ' + stamp)
      .setFontColor('#666666').setFontStyle('italic').setFontWeight('bold');
  }
  return stamp;
}
```

> 기록 위치는 **통합대시보드 A46 셀**(모든 섹션 아래쪽 빈 영역, 안전). 원하는 셀로 바꾸려면 `'A46'`만 수정.

**조치 B:** 아래 항목 2와 합쳐서 `refreshAll`을 통째로 교체 → 다음 블록 참고.

## 항목 2 — 전체 업데이트 시 SNS 채널운영이 매번 재생성됨 (속도저하)

**원인(확정):** `refreshAll`이 매번 `updateSNSReport()`를 호출 → 내부에서 표 전체를 `clear` 후 **수식까지 재생성**. 하지만 SNS 표는 `SUMIFS/COUNTIFS` **수식**이라 한 번 깔면 자동 재계산됨 → 매번 재생성은 불필요한 낭비.

**조치:** `updateSNSReport`에 "이미 만들어졌으면 건너뛰기" 가드를 넣고, `refreshAll`은 재생성 없이 호출.

### (1) `refreshAll` 통째로 교체 (항목 1·2 둘 다 반영)

```javascript
function refreshAll() {
  const ui = SpreadsheetApp.getUi();
  try {
    fetchGA4Daily();
    updateKPISummary();
    updateChannelMatrixWithGA4();
    updateSNSReport({ forceRebuild: false, showAlert: false }); // 이미 있으면 재생성 생략(속도↑)
    repairSNSMonthlySummaries(false);
    updateKakaoInquiryCoverage(false);
    addTimeSeriesChart();
    const stamp = recordLastRefresh_();        // 최근 업데이트 시각 기록
    ui.alert('✅ 전체 새로고침 완료\n🕐 ' + stamp);
  } catch (e) {
    ui.alert('❌ 오류: ' + e.message);
  }
}
```

### (2) `updateSNSReport` — 함수 첫 부분만 교체

기존:
```javascript
function updateSNSReport() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sh = ss.getSheetByName('통합대시보드');
  sh.getRange('A28:Z35').breakApart();
```
↓ 아래로 교체:
```javascript
function updateSNSReport(opts) {
  opts = opts || {};
  const forceRebuild = opts.forceRebuild !== false; // 기본 true (메뉴/수동 실행 시 재생성)
  const showAlert = opts.showAlert !== false;       // 기본 true
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sh = ss.getSheetByName('통합대시보드');
  // 이미 표가 있고 강제 재생성이 아니면 → 재생성 생략 (수식 자동 재계산)
  if (!forceRebuild && sh.getRange('A28').getValue() === '★ SNS 채널 운영') {
    return;
  }
  sh.getRange('A28:Z35').breakApart();
```

### (3) `updateSNSReport` 맨 끝 알림 한 줄 교체

기존:
```javascript
  SpreadsheetApp.getUi().alert('✅ SNS 보고표 갱신 — E28 드롭다운으로 기간 별도 설정 (채널 매트릭스와 분리)');
```
↓
```javascript
  if (showAlert) SpreadsheetApp.getUi().alert('✅ SNS 보고표 갱신 — E28 드롭다운으로 기간 별도 설정 (채널 매트릭스와 분리)');
```

> 효과: 전체 새로고침 때 SNS 표는 **재생성 안 함**(데이터는 수식으로 최신 유지). SNS 표를 진짜로 다시 그리고 싶을 땐 `updateSNSReport()` (인자 없이) 직접 실행하면 됨.

### 같은 속도 문제 — 참고 (사장님 판단)

`updateKPISummary`, `updateChannelMatrixWithGA4`도 **매 새로고침마다 표 전체를 재생성하고 각각 알림창**(`alert`)을 띄움 → SNS와 동일한 속도저하·클릭 유발. 원하시면 똑같은 "가드 + showAlert" 패턴을 두 함수에도 적용해 드림. (이번엔 SNS만 요청하셔서 SNS만 반영함.)

## 항목 4 — 광고소재 시트: 사용법을 K열 → 3행, 3행까지 고정

**확인된 사실:**
- `광고소재` 시트의 `🔗 사용법` 블록(드라이브 업로드 6단계)은 **Apps Script가 만드는 게 아님.** 전체 `.gs` 어디에도 이 시트 생성 코드 없음 → **사장님이 K열에 수기로 넣은 것.** (meta-sync.gs가 만드는 건 이름이 다른 `메타_소재` 시트.)
- 그래서 "코드 한 줄 고치면 끝"이 아니라 **시트 레이아웃 변경**임.
- MCP(구글 드라이브 연동)로 시트 **내용은 읽을 수 있으나**, 셀 이동·행 고정 같은 **구조 변경은 MCP로 불가**(읽기 전용). → 실제 변경은 **(a) 수동 1분 작업** 또는 **(b) 내가 Chrome로 시트 직접 조작**만 가능.

### (a) 수동 절차 (가장 안전, 1분)

1. `광고소재` 시트에서 K열의 `🔗 사용법` 블록 셀들을 드래그 → 잘라내기(Ctrl+X).
2. 데이터 위에 **빈 행 1개 삽입**: 데이터 첫 행 머리글(예: `채널/캠페인/소재명…`) **위 또는 아래**에 행 추가해서 3행을 비움.
   - 권장 구조: **1행=제목, 2행=열 머리글, 3행=사용법 배너, 4행~=데이터**.
3. 비운 3행에서 A3부터 데이터 폭만큼(예: A3:I3) **셀 병합** → 거기에 사용법 텍스트 붙여넣기(가로 배너). 글자 잘리면 행 높이 키우거나 `서식 → 줄바꿈`.
4. `보기 → 고정 → 행 3개까지` 선택 → 스크롤해도 사용법이 항상 위에 보임.
5. 비워진 K열은 그대로 두면 됨(이제 데이터 영역으로 재사용 가능).

### (b) 내가 Chrome로 직접 (원하면)

"Chrome로 광고소재 시트 직접 고쳐줘"라고 하면, 내가 시트를 열어 **실제 행/열 구조를 눈으로 확인한 뒤** 위 (a)를 정확히 적용함. (블라인드 스크립트는 수기 데이터 손상 위험이 있어 권장 안 함.)
