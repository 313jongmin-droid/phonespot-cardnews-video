# Looker Studio 대시보드 셋업 가이드

> **시각화 레이어.** 데이터 입력·관리는 그대로 「폰스팟 광고운영 관리대장」 시트에서. Looker Studio는 그 위에 보기 좋게 얹는 것.

---

## 왜 Looker Studio

- **무료** (구글 계정으로 즉시 사용)
- 데이터 소스 = 시트 직접 연결 (수동 sync 없음, 시트 바꾸면 자동 반영)
- KPI 카드·차트·필터 **드래그앤드롭**
- 모바일 자동 반응형 → 외근 중 폰으로 확인 가능
- 부사수 공유 = 시트 권한 그대로 사용
- TV/모니터에 띄워두기 좋음

---

## 단계 1 — Looker Studio 보고서 생성 (5분)

1. https://lookerstudio.google.com 접속 (313jongmin@gmail.com 로그인)
2. 좌상단 **+ 만들기** → **보고서**
3. 데이터 소스 추가 화면 → **Google Sheets** 선택
4. **「폰스팟 광고운영 관리대장」** 검색 → 선택
5. **워크시트**: 일단 `통합대시보드` 또는 채널별 시트 (아래 "데이터 소스 매핑" 참조)
6. **첫 행을 헤더로 사용** 체크
7. 우측 하단 **추가** → 빈 보고서 생성

---

## 단계 2 — 데이터 소스 매핑

각 페이지마다 다른 시트 연결. 1개 보고서에 여러 데이터 소스 추가 가능.

| 페이지 | 데이터 소스 | 시트 탭 | 사용 컬럼 |
|--------|-----------|--------|---------|
| 개요 | 통합대시보드 | 통합대시보드 | (참고용. 가공된 값) |
| 채널별 효율 | 메타/구글/네이버/카카오/당근 | 각 채널 시트 | A(날짜), E(노출), F(클릭), G(지출), H(문의수) |
| SNS 운영 | 스레드/인스타/유튜브/틱톡 | 각 SNS 시트 | A(날짜), E(조회수), F(좋아요), G(팔로워) |
| 문의/개통 | 문의접수 | 문의접수 | A(날짜), C(개통여부), D(유입채널) |
| 트래픽 | GA4_자동 | GA4_자동 | date, sessionSource, eventName, sessions |

데이터 소스 추가: 보고서 편집 화면 → **데이터 추가** → Google Sheets → 시트 탭 선택

## 단계 3 — 페이지 구성 안

### 📄 페이지 1 — 개요 (Overview)

**목적**: 한눈에 현재 상태 파악. TV 띄워두는 용도.

**구성**:
```
┌─────────────────────────────────────────────────┐
│ 🏷 폰스팟 광고운영 대시보드      [날짜 필터: 30일] │
├─────────────────────────────────────────────────┤
│ [이번달 광고비]  [30일 CPL]  [30일 ROAS]  [출처미상%] │
│   1.6M원         70K원       1.18x       83.3%   │
├─────────────────────────────────────────────────┤
│ 📊 채널별 30일 광고비 (도넛)│ 📈 일별 광고비 추세 │
├─────────────────────────────────────────────────┤
│ 📊 카톡클릭 채널 분포        │ 📈 일별 문의 추세  │
└─────────────────────────────────────────────────┘
```

**컴포넌트**:
- KPI 카드 4개 (스코어카드)
- 도넛 1: 채널별 광고비 비중
- 도넛 2: 카톡클릭 채널 분포 (GA4_자동에서)
- 시계열 1: 일별 광고비 (채널별 색상)
- 시계열 2: 일별 문의수

### 📄 페이지 2 — 채널별 효율

**목적**: 채널 비교, 효율 좋은 채널 식별.

**구성**:
- 상단 필터: 기간 (어제/7일/30일/사용자 정의)
- 채널 매트릭스 표: 채널 / 노출 / 클릭 / 소진 / CPL / 카톡클릭 / 카톡당CPC / 평가
- 막대 그래프: 채널별 광고비
- 막대 그래프: 채널별 카톡클릭
- 산점도: x=광고비, y=카톡클릭, 크기=문의수 (효율 한눈에)

### 📄 페이지 3 — SNS 운영

**목적**: 스레드/인스타/유튜브/틱톡 성과 비교.

**구성**:
- 채널별 KPI 카드 (스레드/인스타/유튜브/틱톡 4개) — 이번달 포스트수, 총 조회수
- 시계열: 채널별 일별 조회수
- 막대: 채널별 평균 조회수
- 상위 5 콘텐츠 표 (조회수 기준)

### 📄 페이지 4 — 문의/개통

**목적**: funnel 분석. 광고 → 문의 → 개통.

**구성**:
- KPI: 문의수, 출처확인수, 개통수, 개통률
- 막대: 채널별 문의 / 개통 / 출처미상
- 시계열: 일별 문의 vs 개통
- 산점도/표: 채널별 CPL vs 개통률

## 단계 4 — 필터·컨트롤 추가

### 페이지 상단 (전역)
- **날짜 범위 컨트롤**: 어제 / 최근 7일 / 최근 30일 / 사용자 정의
- 페이지 모든 차트가 이 날짜에 반응

### 채널 페이지 사이드
- 채널 드롭다운 (메타/구글/네이버/카카오/당근)
- 다중 선택 가능

## 단계 5 — 색상·디자인

### 채널별 색상 통일

| 채널 | 색 코드 | 의미 |
|------|--------|------|
| 메타 | `#378ADD` 파랑 | Facebook 파랑 |
| 구글 | `#639922` 초록 | Google 녹색 |
| 네이버 | `#1D9E75` 진녹 | 네이버 컬러 |
| 카카오 | `#FAC775` 노랑 | 카카오 노랑 |
| 당근 | `#BA7517` 주황 | 당근 컬러 |

→ 차트 → 스타일 → 색상별로 지정

### SNS 채널 색상

| 채널 | 색 코드 |
|------|--------|
| 스레드 | `#000000` (스레드 블랙) |
| 인스타 | `#E1306C` (인스타 핑크) |
| 유튜브 | `#FF0000` (유튜브 빨강) |
| 틱톡 | `#69C9D0` (틱톡 시안) |

### 폰트
- 헤더: Noto Sans KR Bold
- 본문: Noto Sans KR Regular
- KPI 카드 큰 숫자: 32~40pt

### 배경
- 깔끔하게 흰색 또는 매우 연한 회색 (#F8F9FA)
- 카드: 흰색 + 미세 그림자
- 다크모드 별도 X (보고용은 라이트 모드가 표준)

## 운영 시 주의

### 데이터 갱신
- 시트 데이터는 **즉시 반영** (refresh 버튼 클릭 시)
- 자동 새로고침: 보고서 우측 상단 ⟳ 아이콘에서 간격 설정 가능 (최소 1시간)

### 시트 컬럼 추가/변경 시
- Looker Studio 데이터 소스 → 필드 새로고침 필요
- 보고서 편집 → 데이터 소스 → ⟳ → 변경된 필드 확인

### 계산 필드
- 시트에 없는 계산 (CPL=지출/문의, ROAS 등)은 Looker에서 계산 필드로 추가
- 예: `CPL = SUM(지출) / SUM(문의수)`
