# ads_kt/ — KT다이렉트샵 광고운영 관리 허브

> **AI 진입점.** KT다이렉트샵 광고운영 관련 작업 시작 시 가장 먼저 읽는 파일.

---

## 이 폴더는 뭐냐

**KT다이렉트샵** 브랜드의 광고운영 + KPI 관리. `phonespot_cardnews/ads/` (폰스팟)와 **완전 분리** 운영.

### 브랜드 컨셉 (핵심)

| 항목 | 폰스팟 (ads/) | KT다이렉트샵 (ads_kt/) |
|------|--------------|---------------------|
| 컨셉 | 휴대폰 성지 판매점 | **KT 공식 인증 대리점** |
| 디자인 톤 | 자유 (색깔별 배너 등) | **KT 본사 브랜드 가이드 준수** |
| 광고 메시지 | "가격 싸게", "성지" | "공식 인증", "안전", "KT 직영" |
| 타겟 | 손품·발품 가격 비교층 | KT 사용자, 공식·신뢰 추구 |
| 광고 정책 | 자유 | **KT 가이드라인 위반 시 계약 위험** |
| 채널 믹스 | 당근·메타·카카오 중심 | **네이버 검색·구글 검색 중심** (공식 키워드) |

→ 두 브랜드 절대 통합 X. 통합 모니터링이 필요하면 회사 차원 Looker Studio에서 두 데이터소스 묶기.

---

## 폰스팟 ads/ 와 다른 점

이 폴더는 **폰스팟 ads/ 의 사본**을 기반으로 시작:
- 시트 사본 (Apps Script 코드 그대로 + 시트 ID만 변경)
- 폴더 구조 동일 (`code/`, `data/`, `bridges/`, `integrations/`)
- **30분 정리** 후 KT다이렉트샵 운영 시작

### KT다이렉트샵 시작 시 30분 정리 항목

| 항목 | 폰스팟에 있는데 KT다이렉트샵 없음 |
|------|--------------------------------|
| 시티마켓 클릭 추적 | 제거 (KT는 시티마켓 안 씀) |
| 리틀리 페이지 | 제거 (KT 직접 랜딩) |
| 당근 채널 시트 | 제거 (KT 공식은 당근 못 씀) |
| 카카오 채널 시트 | 제거 또는 유지 (KT 정책 확인) |
| 미운영 SNS 채널 | 일단 다 제거, 운영 시 추가 |
| Code.gs 상수 | `GA4_PROP_ID`, 시트 ID 등 KT용으로 교체 |

### KT다이렉트샵 추가/강조 항목

| 항목 | 신설 또는 강조 |
|------|-------------|
| 사전승낙서 관리 | KT 본사 발급 사전승낙서 추적 |
| 키워드 화이트리스트 | KT 가이드라인 허용 키워드만 |
| 광고 소재 검수 이력 | KT 본사 승인 / 미승인 / 수정 요청 |
| 매장별 코드 | 폰스팟 본점/안양/광교 → KT다이렉트샵 본사 가맹 코드 |
