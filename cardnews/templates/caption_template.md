## 0. 자동화 흐름 (가장 먼저 이해)

```
articles/<slug>.json
  ├─ cards          (6장 헤드라인 + body + source)
  ├─ captions_md    (5채널 = 블로그/스레드/인스타/유튜브/틱톡)
  └─ narration_md   (영상 나레이션 단일 본문)
        │
        ▼  run_pngs.bat 실행
        │
output/<slug>/
  ├─ card_*.jpg (18장 = 6장 × 3 사이즈 1080/720/480)
  └─ captions.md  ← captions_md 그대로 + 마지막에 narration_md를 채널 6 으로 자동 append
```

- **JSON 작성 단계에서 captions_md + narration_md 둘 다 박아둠.**
- `narration_md` 는 run_pngs 시 자동으로 `captions.md` 최하단에 채널 6 (영상 나레이션) 으로 추가됨. 별도 작성 X.

## 1. 공통 토큰 (모든 채널 필수)

| 토큰 | 치환값 | 사용처 |
|---|---|---|
| `{LITTLY}` | `litt.ly/phonespot` | 모든 채널 본문 + 프로필 링크 표기 |
| `{PRECON_URL}` | 사전승낙서 공식 URL | 모든 채널 최하단 `[사전승낙서] {PRECON_URL}` |

- renderer가 자동 치환. **JSON에서 절대 URL을 직접 박지 말 것.**
- 한국 통신판매 법령상 SNS 광고성 게시물은 사전승낙서 URL 필수 = **모든 채널 마지막 줄 강제**.
- 나레이션 음성에는 URL 안 읽음 (narration_md 에서 `{LITTLY}` `{PRECON_URL}` 사용 ❌).

## 5. 6장 카드 구조 (모든 라인 공통)

| 카드 # | 역할 | 패턴 |
|---|---|---|
| 1 | 후크 (Hook) | 질문·놀라움·시즌 후크. "★ 모르는 사실 / 피해 규모 / 시작일" |
| 2 | 핵심 사실 1 | 토픽의 가장 단단한 사실 (출시일·통계·정의) |
| 3 | 핵심 사실 2 | 두 번째 사실 (수법·기능·예시) |
| 4 | 체크·차단·기능 N | "받을 수 있는 N가지 / 점검 N가지 / 차단 N가지" |
| 5 | 행동·신고·신청 채널 | "신청처 / 신고 / 골든타임 / 5분 대응" |
| 6 | 매장 CTA | "광교점 폰스팟 [구체 도움] / 1:1 상담" |

### 카드 헤드라인 룰

- 형식: `[키워드 1줄]\n<span class="hl">강조</span> [키워드 2줄]`
- `\n` = 줄바꿈 (자바스크립트 처리), `<span class="hl">` = 오렌지(#F74B0B) 강조
- 카드 1 헤드라인은 **모든 채널 공통**이라 명사형 + 신뢰 톤 유지
- 캡션 첫 줄에서만 채널별 톤 변형 (3번 룰)

### 카드 body 룰

- 길이: 100~140자 (카드 안에서 가독성)
- 톤: 존댓말 + 단정형. "~합니다 / ~하시기 바랍니다"
- 출처: `source` 필드에 짧게 1줄 (`출처: <매체·기관>`)
- 마지막 카드(6) 의 `source` 만 `Phonespot 광교점`

- 한 매체는 1회만. 같은 통계 반복 인용해도 출처 묶어서 1번.

### `source_line` (JSON 최상위 필드)

## 9. JSON 작성 안전 룰 (★ 사고 방지)

`captions_md` `narration_md` 같은 큰 멀티라인 텍스트는 JSON 문자열 안에 `\n` (백슬래시 n) 으로 인라인 박힘. 안전한 박기 방법:

1. **Write 도구로 한 번에 전체 JSON 작성** — Edit 으로 큰 필드 부분 수정 ❌ (JSON 닫는 `}` 누락 사고)
2. **반드시 Write 후 json.load 검증 가능한 상태로 둘 것**
3. 큰 변경 시 bash + `json.dump` 패턴 권장 (워크스페이스 bash 사용 가능 시):
   ```python
   import json
   data = json.load(open(p, 'r', encoding='utf-8'))
   data['captions_md'] = new_value
   json.dump(data, open(p, 'w', encoding='utf-8'), ensure_ascii=False, indent=2)
   ```
4. 위 방식 불가 시 — Write 로 전체 다시 박기 (덮어쓰기)
