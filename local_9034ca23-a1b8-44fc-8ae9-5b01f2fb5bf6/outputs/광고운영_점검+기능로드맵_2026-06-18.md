## PART 2 — 개선 기능 (이미 있는 걸 더 효율·효과적으로)

우선순위순. [난이도] 동봉.

1. **문의 유입채널 자동/강제화** [중] — 최우선. 카톡/전화 유입 시 GA4 utm_source로 채널 자동 추정해 문의접수 D열 프리필, 또는 입력 드롭다운 필수화 + 매일 "채널 미입력 N건" 텔레그램 알림. 이게 고쳐져야 광고 ROI가 숫자로 보임.
2. **침묵 실패 → 헬스체크 + 알림** [하] — 동기화 후 "채널별 최근 N일 GA4 매칭 0행 / 0데이터 / 0광고그룹" 자동 점검 1줄을 동기화_로그·텔레그램에. 네이버 0행 같은 사고를 자동 포착.
3. **utm 슬러그 drift 방지** [중] — autoDiscover가 새 광고그룹 추가 시 GA4 실측 utm_campaign 목록을 C열 드롭다운으로 제시 → `region` vs `region_keyword` 같은 오타·불일치 차단. (네이버 region/mobile_carrier/powerlink 현재 불일치)
4. **통합대시보드 야간 자동 새로고침** [하] — refreshAll을 매일 새벽 트리거에 등록(현재 06-11 이후 수동 미실행). 인사이트 MD·추세·스탬프가 자동 최신화.
5. **컬럼 위치 결합 → named range/헤더명 기반** [중] — `insertColumnBefore` 한 번에 타 시트 SUMIFS가 밀려 깨지는 구조(오늘 사고 원인). 명명 범위로 위치 의존 축소.
6. **구조변경 마이그레이션 안전 절차 표준화** [하] — 컬럼 변경 함수 끝에 sync 재호출 자동 + "다음 단계 전체 새로고침" 안내 강제.
7. **광고그룹 추이 자동 갱신** [하] — 현재 수동 토글 후 🔄. 야간에 마지막 선택 기준 자동 갱신.
8. **B1 시트 read 효율화** [하] — read_file_content가 마크다운 이스케이프(`\_`)로 JSON 깨뜨림. export를 CSV로도 저장하거나, "채널별 최신일자·매칭여부"만 담은 작은 헬스 JSON을 매일 저장 → 클로드가 1파일로 전 채널 상태 파악.
9. **리포 위생 정리** [하] — 미커밋 132개 + governance gate(`.githooks/`) 활용해 working tree 한 번 정리(다음 stash 사고 예방).
