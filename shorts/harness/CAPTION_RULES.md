# CAPTION_RULES

> 청크 분할·자막·나레이션 규칙. 사람이 보기 자연스러운 한국어 호흡 우선.

---

## TTS vs 자막 — 두 트랙

- **TTS(나레이션)** = 풀문장 기반. 자연스럽게 읽히는 한국어 문장 단위.
- **자막** = 1.5~2초 안에 한 호흡으로 읽을 분량으로 잘림.

두 트랙은 필요시 분리. TTS는 매끄러운 한 문장, 자막은 그 문장을 1~3개 청크로 분할.

---

## 청크 분할 — 의미 단위 우선

자막 청크 분할은 의미·호흡 단위로. 우선순위:

1. **문장 끝** (마침표·물음표·느낌표)
2. **쉼표/절 끝** (쉼표·콜론·세미콜론)
3. **조건절/종속절** ("~면", "~인데", "~지만", "~라서")
4. **나열 구조** ("A·B·C 중에", "첫째 / 둘째")

길이 기준:
- 한 청크 = 한국어 12~18자 권장 (1.5~2초 호흡)
- 최대 25자 넘기지 말 것

---

## 영문/숫자 약어 — 단독 분리 금지

다음 약어들은 **앞 단어와 묶어서** 한 청크에. 단독 청크로 띄우면 자막이 뚝 끊겨 어색.

- OS·앱 이름: `iOS`, `iOS 26.6`, `One UI`, `Android`
- 제품 모델: `S25`, `S26`, `Z 폴드 7`, `갤럭시 S26`
- 결제·통신: `NFC`, `eSIM`, `5G`, `LTE`, `RCS`
- 매체·기관: `ATM`, `KISA`, `WWDC`, `FCC`
- 통신사: `SKT`, `KT`, `LGU+`
- 가격·금액: `35만원`, `100유로`, `5조 달러`
- 날짜: `5월 27일`, `6/8`, `2026년`

좋은 예:
```
"iOS 26.6 첫 베타에서" / "연락처 차단 2만 개 알림"
```

나쁜 예 (단독 분리 금지):
```
"iOS" / "26.6 첫 베타에서" / "연락처 차단" / "2만 개" / "알림"
```

## 자막 시각 규칙

- **하단 영역에 크게** 표시 (영상 하단 30% 안)
- 폰트: Pretendard Bold 이상
- 글자 색: 흰색 + 어두운 stroke 또는 box 배경
- 한 줄에 가능한 모두 표시. 두 줄 분할은 의미가 정말 끊길 때만.
- 강조 단어(숫자·날짜·금액·제품명·OS명)는 오렌지 `#F74B0B` 또는 노란 형광으로 1~2개만 강조

---

## 자동 강조 룰 (build_script.py가 적용)

자막에 자동 강조 적용 대상:

- 숫자·수량: `15장`, `3건`, `최대 5조`
- 금액·가격: `35만 원`, `100유로`, `5조 달러`
- 날짜·기간: `5월 27일`, `6/8`, `D-7`
- 제품명: `갤럭시 S26`, `아이폰18`, `Z 폴드 7`
- OS·플랫폼: `iOS 26.6`, `One UI 8.5`, `WWDC 2026`

⚠ **강조는 청크당 1~2개로 제한**. 너무 많이 강조하면 AI 자동화 티가 나서 매장 신뢰도 떨어짐.

---

## 한국어 자연스러움 — 추가 룰

- **종결어미 다양화** — 매 청크가 "~합니다·~입니다"로 끝나지 않게. "~죠", "~네요", "~거든요" 등 섞기
- **외래어 처리** — 영어로 굳어진 단어(스마트폰·아이폰·갤럭시)는 그대로. 굳이 한글로 번역 X
- **숫자 읽기** — TTS가 잘못 읽는 경우 발음 표기 사용 (예: `2026년` → `이천이십육년`이 자연스러우면 분리)
- **약어 표기** — `WWDC` → 자막엔 그대로, TTS엔 `더블유 더블유 디 씨` 또는 `WWDC` 그대로 읽힘 확인

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 자막이 뚝뚝 끊김 | 청크 분할이 너무 잦음 | `build_script.py`의 max_chunk_chars 늘림 |
| 자막이 너무 길어 잘림 | 청크 1개가 25자 초과 | 의미 단위로 추가 분할 |
| TTS가 약어를 이상하게 읽음 | edge-tts 발음 사전 한계 | shorts_script.json에서 TTS 텍스트만 별도 표기 |
| 자막과 TTS 어긋남 | 두 트랙 분리 안 됨 | shorts_script.json의 `tts_text`와 `caption_text` 분리 필드 활용 |

## TTS와 화면 청크 — 반드시 일치

- TTS 나레이션과 화면 청크는 별도의 요약문으로 만들지 않습니다.
- 화면 청크는 해당 구간 TTS 원문을 순서대로 빠짐없이 나눈 결과여야 합니다.
- 화면에서 마침표는 숨길 수 있지만, 단어를 추가하거나 삭제하거나 바꾸지 않습니다.
- 청크 분할은 문장 끝, 쉼표, 연결 어미, 한국어 호흡 순서로 판단합니다.
- 한 청크는 공백 제외 9~27자를 우선하며, 최대 34자를 넘기지 않습니다.
- 1초 미만으로 지나가는 고아 청크를 만들지 않습니다.
- TTS 발음 사전은 음성 합성 단계에서만 적용합니다. 화면 자막 원문은 바꾸지 않습니다.
- 시각 자료 매핑, 고정 CTA, 원본 이미지 1회 사용, 일러스트 균형 규칙은 그대로 유지합니다.
