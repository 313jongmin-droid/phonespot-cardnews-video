# promo 학습 루프 — 인사이트 → 훅 적용 + 변주 매니페스트 (현행 2026-06-15)

> promo 정본은 `promo/README.md`. 이 문서는 그 §학습 루프 상세(②단계).
> 목적: 데이터로 검증된 후킹 패턴을 promo 첫 줄에 반영하고, 어떤 변주를 뽑았는지 기록해 둔다.

## 루프 4단계 (현재 상태)
1. **학습(인사이트)** — `유튜브_인사이트` 시트(매일 03:40 Top 키워드·후킹 패턴). `메타_인사이트`(시트화 시 합류). ← 이미 동작.
2. **적용(훅 생성)** — promo 대본 첫 줄(훅)을 인사이트 Top 패턴으로 작성 + `후킹:` 타입 선언. ← 이 문서.
3. **발행** — 업로드(수동).
4. **성과 귀속** — 변주별 조회/저장 집계(= C단계, ads/API 연동 필요, 미구현).

→ 현재 ①+② 반자동. ④의 "변주별 성과 자동 판별"은 매니페스트(아래)로 데이터만 먼저 쌓고, C단계에서 닫는다.

## 후킹 타입 8종 (ads 생성기와 동일 분류)
질문형 · 단언형 · 비교형 · 한정형 · 가격강조 · 감성·공감 · 위협형 · FOMO형
- ⚠ promo는 가격 숫자 금지 → "가격강조"는 **투명·정찰 메시지**로만(금액 X).

## 작성 규칙 (② 적용)
1. 대본 쓰기 전 `유튜브_인사이트`(+`메타_인사이트`) Top 후킹 패턴을 본다(sync_sources 자동 전달).
2. 훅(첫 비트) 첫 줄 = 그 Top 패턴을 폰스팟 맥락으로 변형. 1편당 한 가지 패턴에 집중.
3. review MD 머리에 `- 후킹: <타입>` 선언 (예 `- 후킹: 질문형`). 비우면 `(미지정)`으로 기록만 됨.
4. 같은 대본을 preset(showcase/punchy/calm/data)만 바꿔 변주 렌더 → 자동 넘버링으로 쌓임 → 매니페스트가 각 변주를 기록.

## 변주 매니페스트 (④ 대비 데이터)
- 파일: `promo/_manifest.csv` (Excel 호환 utf-8-sig). 렌더 성공할 때마다 1행 자동 추가.
- 기록기: `scripts/promo_manifest.py "<outfile>" <nn> <slug> <preset>` — `run_promo*.bat`이 렌더 직후 자동 호출.
- 컬럼: ts · outfile · nn · slug · preset · hook_pattern · hook_text · styles · music_src · music_start · n_facts.
- 이걸로 나중에 outfile ↔ 업로드 영상 매칭만 하면(C단계) "어떤 후킹/스타일/프리셋이 이겼나" 집계 가능.

## 주의(통계)
- SNS 초기엔 영상별 표본 작음 → "이 훅이 이겼다"가 노이즈일 수 있음. 표본 적을 땐 사람이 판단, 누적되면 C로 자동화.
