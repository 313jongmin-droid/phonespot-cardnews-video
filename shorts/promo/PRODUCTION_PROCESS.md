# 폰스팟 홍보영상 양산 프로세스 점검

> 목적: "이런 영상"을 꾸준히·반복 가능하게 찍어내기 위한 현재 파이프라인 점검 + 표준 절차 + 병목/개선.

## 0. 지금 갖춘 자산 (점검 결과)
- **엔진**: `PromoShort` 디스패처 + `styleMap`(구간별 스타일=디렉터 모드) + 스타일 23종 + `Promo-showcase`.
- **스크립트**: `promo/promo_jeongchalje.json`(정찰제 1편) · 신규용 `promo/_template/promo_TEMPLATE.json`.
- **실행**: `run_promo.bat`(스타일 선택 → public 복사 → edge-tts → Remotion 렌더 → `out/promo/`).
- **가이드**: GUIDE_TYPOGRAPHY / GUIDE_INFOGRAPHIC / GUIDE_BEST_TYPO_AD / STYLE_CATALOG / STYLE_EVAL.
- **백업**: `promo/_backup/`.

## 1. 1편 만드는 표준 절차 (반복 레시피)
1. **기획** — 한 토픽 = 한 메시지. 6비트 구조: opening · hook · fact×3 · cta.
2. **스크립트** — 템플릿 복사 → `promo/<slug>.json`. `caption_chunks`=화면자막(짧게), `tts`=나레이션(자연).
3. **디자인** — `styleMap` 선택(showcase 권장) 또는 단일 스타일.
4. **렌더(PC)** — `run_promo.bat`(또는 Studio `Promo-showcase`) → `out/promo/<slug>.mp4` (1080×1920·Pretendard·나레이션).
5. **편집** — 음악 + 컷 비트싱크 + 전환 효과음 + 컬러 그레이드.
6. **QA** — 규제(숫자·비방), 자막 가독성, yuv420p·길이·용량.
7. **게시** — 캡션 + 해시태그 + "영상 제작 문의" 링크.

## 2. 병목 / 갭 (솔직하게)
- **콘텐츠(스크립트)가 수작업.** 토픽만 있으면 초안 자동 생성이 없음 → 템플릿으로 완화. (카드뉴스 자동화처럼 토픽→스크립트 생성기까지 가능.)
- **렌더는 PC 전용.** 샌드박스는 Chromium 차단으로 Remotion 렌더 불가 → 양산은 PC 배치로 돌려야 함.
- **`run_promo.bat`은 1편씩.** 여러 편 일괄 렌더(배치) 없음.
- **styleMap 선택이 매번 판단.** 프리셋이 없어 매번 고민.
- **음악·효과음·그레이드가 편집 수작업** — 양산 속도를 가장 크게 좌우.
- **규제 검수 수작업**(숫자·경쟁사 비방).
- **브랜드 공통값(채널·kakao·litt·location)을 스크립트마다 반복 입력** → 분리 가능.

## 3. 양산을 위한 개선 (우선순위)
1. **토픽 뱅크 + 템플릿**(템플릿 완료): 주 N편 초안 미리 쌓기.
2. **styleMap 프리셋 3~4종** 정의(showcase / punchy / calm / data) → 선택만.
3. **배치 러너** `run_promo_batch.bat`: `promo/*.json` 전부 순회 렌더 → `out/promo/`.
4. **브랜드 공통값 분리**(채널·kakao·litt·location 한 파일) → 스크립트는 메시지만.
5. **편집 프리셋**: 음악 트랙 풀 + 전환 효과음 세트 + 그레이드 고정 → 편집 시간 단축.
6. **QA 체크리스트 1장**(규제+기술) 고정.
7. **게시 캘린더**: 정보성(카드뉴스) 다수 + 홍보 소수, 주간 고정 슬롯.

## 4. 현실적 처리량(편당)
- 스크립트+styleMap: **10~20분**(템플릿 사용 시).
- 렌더(PC): **2~3분**.
- 편집(음악·비트싱크): **20~40분** ← 양산 속도의 핵심 변수.
→ **편집 표준화**(음악/효과음/그레이드 프리셋)가 가장 큰 레버.
