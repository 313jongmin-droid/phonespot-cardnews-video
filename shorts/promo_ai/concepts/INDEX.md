## 트랙(track)
- **ad** = 광고용(paid): 전환·CTA 강, 15s 고정, 측정 = CTR·카톡전환

## 성과 기입 + 학습 규칙 (★ 2026-06-24)

1. **측정축 분리**: viral = 조회·완료율·공유·저장·댓글(CTR ❌). ad = CTR·카톡전환. 같은 컬럼에 섞지 말 것.
2. **환류**: 신규 viral 빌드 시 표 B Read → ① 후킹타입별 평균 공유·저장 집계 → 상위 타입 우선 채택 ② 제품·수법 슬롯도 고성과 우선. caption_template 유튜브 후킹 패턴(외부 학습)과 병행.
3. ★ **표본 부족 정직 룰**: 표 B 데이터 5편 미만이면 "표본 부족 → 슬롯 가중치 보류, 균등 진행" 1줄 명시(과적합 방지). 5편↑부터 가중치 적용.
4. **20편↑이면 시트 승격** → Phase 1(유튜브 자동 sync)·2(인스타 릴스 인사이트 자동분석)로 자동화. 설계 정본 = 이 파일 + INSIGHTS_LOOP.md 패턴 재활용.
